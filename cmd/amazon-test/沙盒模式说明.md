# 🧪 Amazon SP-API 沙盒模式说明

## ✅ 沙盒模式已启用

配置文件中已默认启用沙盒模式，可以安全测试！

## 📋 什么是沙盒模式？

沙盒模式是 Amazon 提供的测试环境，让你可以：

✅ **安全测试** - 不会创建真实的商品
✅ **无限测试** - 不消耗 API 配额
✅ **快速验证** - 验证代码逻辑是否正确
✅ **零风险** - 不会影响你的 Amazon 账号

## ⚙️ 当前配置

查看 `config/config-dev.yaml`：

```yaml
amazon:
  spapi:
    sandbox: true  # ← 沙盒模式已启用
```

## 🔄 如何切换环境

### 开发测试（当前）
```yaml
sandbox: true  # 使用沙盒环境
```

### 正式上架
```yaml
sandbox: false  # 使用生产环境
```

**⚠️ 警告：** 切换到生产环境后，所有操作会影响真实数据！

## 📊 沙盒 vs 生产环境

| 特性 | 沙盒模式 | 生产环境 |
|------|---------|---------|
| 影响真实数据 | ❌ 否 | ✅ 是 |
| 创建真实商品 | ❌ 否 | ✅ 是 |
| API 配额消耗 | ❌ 否 | ✅ 是 |
| 测试次数 | ♾️ 无限 | ⚠️ 有限 |
| 适用场景 | 开发测试 | 正式上架 |

## 🎯 推荐流程

### 阶段 1: 沙盒测试 ✅（当前阶段）
```yaml
sandbox: true
```
- ✅ 验证 API 调用
- ✅ 测试错误处理

## 💡 使用建议

1. **始终先在沙盒测试**
   - 新功能开发时使用沙盒
   - 确保代码逻辑正确
   - 验证错误处理完善

2. **充分测试后再切换**
   - 在沙盒环境测试所有场景
   - 确认功能完全正常
   - 准备好监控和回滚方案

3. **生产环境谨慎操作**
   - 小规模测试
   - 监控错误和异常
   - 随时准备回滚

## ⚠️ 重要提醒

1. **沙盒和生产使用相同凭证**
   - Client ID
   - Client Secret
   - Refresh Token

2. **沙盒数据是模拟的**
   - 返回的响应可能与生产环境不同
   - 某些验证规则可能略有差异

3. **切换环境需要重启**
   - 修改配置后需要重启程序
   - 确认端点已正确切换
