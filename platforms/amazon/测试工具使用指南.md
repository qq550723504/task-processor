# Amazon 商品上传测试工具 - 完整使用指南

## ✅ 工具已就绪

Amazon 商品上传测试工具已完全配置好，支持沙盒模式测试！

## 🎯 快速开始（3 步）

### 第 1 步：获取 Amazon SP-API 凭证

1. 登录 [Amazon Seller Central](https://sellercentral.amazon.com/)
2. 进入：**设置 > 用户权限 > 开发者中心**
3. 创建新应用程序
4. 获取以下凭证：
   - Client ID
   - Client Secret
   - Refresh Token

详细步骤：`config/amazon-spapi-setup.md`

### 第 2 步：配置凭证

编辑 `config/config-dev.yaml`，填入你的凭证：

```yaml
amazon:
  spapi:
    enabled: true
    sandbox: true  # 沙盒模式（推荐用于测试）
    region: "us-east-1"
    marketplaceID: "ATVPDKIKX0DER"
    clientID: "你的 Client ID"        # ← 填写这里
    clientSecret: "你的 Client Secret" # ← 填写这里
    refreshToken: "你的 Refresh Token" # ← 填写这里
    defaultFulfillmentType: "FBM"
    defaultCondition: "New"
```

### 第 3 步：运行测试

```cmd
go run ./cmd/amazon-test
```

## 🧪 沙盒模式（推荐）

### 什么是沙盒模式？

沙盒模式是 Amazon 提供的测试环境：

✅ **不会创建真实商品** - 所有操作都是模拟的
✅ **不消耗 API 配额** - 可以无限次测试
✅ **零风险测试** - 不会影响你的 Amazon 账号
✅ **快速验证** - 验证代码逻辑是否正确

### 当前状态

配置文件中已默认启用沙盒模式：
```yaml
sandbox: true  # ← 沙盒模式已启用
```

详细说明：`cmd/amazon-test/沙盒模式说明.md`

### 常见错误

#### 1. 凭证未配置
```
❌ 配置检查失败: amazon.spapi.clientID 未配置
```
**解决：** 在配置文件中填写真实凭证

#### 2. 认证失败
```
❌ 认证失败: invalid_client
```
**解决：** 检查 Client ID 和 Client Secret 是否正确

#### 3. API 速率限制
```
❌ 上传失败: API rate limit exceeded
```
**解决：** 等待 1-2 分钟后重试

## ⚠️ 注意事项

1. **沙盒和生产使用相同凭证**
   - 不需要单独申请沙盒凭证
   - 通过配置切换环境

2. **沙盒数据是模拟的**
   - 返回的响应可能与生产环境不同
   - 用于验证代码逻辑，不是真实数据

3. **切换环境需要重启**
   - 修改配置后需要重启程序
   - 确认端点已正确切换

4. **生产环境谨慎操作**
   - 充分测试后再切换
   - 小规模验证
   - 监控错误率
