# 日志系统迁移指南

本指南帮助你将现有代码迁移到新的统一日志系统。

## 迁移步骤

### 1. 移除直接使用 logrus 全局方法

**旧代码**：
```go
import "github.com/sirupsen/logrus"

logrus.Info("任务开始")
logrus.Errorf("任务失败: %v", err)
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("my_component")
log.Info("任务开始")
log.WithError(err).Error("任务失败")
```

### 2. 统一 logger 实例创建

**旧代码**：
```go
type MyService struct {
    // 没有logger字段
}

func (s *MyService) DoSomething() {
    logger := logrus.WithField("component", "MyService")
    logger.Info("doing something")
}
```

**新代码**：
```go
import (
    "github.com/sirupsen/logrus"
    "task-processor/internal/core/logger"
)

type MyService struct {
    logger *logrus.Entry
}

func NewMyService() *MyService {
    return &MyService{
        logger: logger.GetGlobalLogger("my_service"),
    }
}

func (s *MyService) DoSomething() {
    s.logger.Info("doing something")
}
```

### 3. 替换 fmt.Printf 调试输出

**旧代码**：
```go
import "fmt"

fmt.Printf("调试信息: %v\n", data)
fmt.Println("处理完成")
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("my_component")
log.WithField("data", data).Debug("调试信息")
log.Info("处理完成")
```

### 4. 统一字段命名

**旧代码**：
```go
logger.WithFields(logrus.Fields{
    "taskId":    taskID,      // 驼峰命名
    "ProductID": productID,   // 大写开头
    "tenant_id": tenantID,    // 下划线
}).Info("处理任务")
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("task_handler")
log.WithFields(logrus.Fields{
    logger.FieldTaskID:    taskID,
    logger.FieldProductID: productID,
    logger.FieldTenantID:  tenantID,
}).Info("处理任务")
```

### 5. 使用 Context 传递 Logger

**旧代码**：
```go
func ProcessTask(ctx context.Context, task Task) error {
    logger := logrus.WithField("task_id", task.ID)
    logger.Info("开始处理")
    
    // 调用其他函数时无法传递logger
    result := doSomething(ctx, task)
    return nil
}

func doSomething(ctx context.Context, task Task) Result {
    // 需要重新创建logger
    logger := logrus.WithField("task_id", task.ID)
    logger.Debug("执行操作")
    return Result{}
}
```

**新代码**：
```go
import "task-processor/internal/core/logger"

func ProcessTask(ctx context.Context, task Task) error {
    log := logger.GetGlobalLogger("task_processor").WithField("task_id", task.ID)
    
    // 将logger添加到context
    ctx = logger.WithLogger(ctx, log)
    
    log.Info("开始处理")
    result := doSomething(ctx, task)
    return nil
}

func doSomething(ctx context.Context, task Task) Result {
    // 从context获取logger
    log := logger.FromContext(ctx, "helper")
    log.Debug("执行操作")  // 自动包含task_id
    return Result{}
}
```

### 6. 错误日志规范化

**旧代码**：
```go
if err != nil {
    logrus.Errorf("操作失败: %v", err)
    return err
}
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("my_service")

if err != nil {
    log.WithError(err).WithFields(logrus.Fields{
        logger.FieldOperation: "fetch_data",
        "url": apiURL,
    }).Error("操作失败")
    return fmt.Errorf("获取数据失败: %w", err)
}
```

### 7. 使用日志辅助工具

**旧代码**：
```go
start := time.Now()
logger.Info("开始操作")

err := doSomething()
duration := time.Since(start)

if err != nil {
    logger.WithFields(logrus.Fields{
        "duration_ms": duration.Milliseconds(),
    }).Errorf("操作失败: %v", err)
    return err
}

logger.WithFields(logrus.Fields{
    "duration_ms": duration.Milliseconds(),
}).Info("操作完成")
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("my_service")
helper := logger.NewLoggerHelper(log)

err := helper.LogOperation("do_something", func() error {
    return doSomething()
})
```

### 8. 进度日志优化

**旧代码**：
```go
for i, item := range items {
    // 每次都记录日志
    logger.Infof("处理进度: %d/%d", i+1, len(items))
    processItem(item)
}
```

**新代码**：
```go
import "task-processor/internal/core/logger"

log := logger.GetGlobalLogger("batch_processor")
helper := logger.NewLoggerHelper(log)

for i, item := range items {
    // 每100个或最后一个记录一次
    if (i+1)%100 == 0 || i == len(items)-1 {
        helper.LogProgress(i+1, len(items), "批处理进度")
    }
    processItem(item)
}
```

## 具体文件迁移示例

### 示例 1: task_handler.go

**迁移前**：
```go
package temu

import (
    "github.com/sirupsen/logrus"
)

type TaskHandler struct {
    processor *TemuProcessor
    logger    *logrus.Entry
}

func NewTaskHandler(processor *TemuProcessor) *TaskHandler {
    return &TaskHandler{
        processor: processor,
        logger: logrus.WithFields(logrus.Fields{
            "component": "TaskHandler",
            "platform":  "temu",
        }),
    }
}

func (h *TaskHandler) ProcessTask(ctx context.Context, task model.Task) error {
    h.logger.Infof("开始处理任务: ID=%d, ProductID=%s", task.ID, task.ProductID)
    // ...
}
```

**迁移后**：
```go
package temu

import (
    "task-processor/internal/core/logger"
    "github.com/sirupsen/logrus"
)

type TaskHandler struct {
    processor *TemuProcessor
    logger    *logrus.Entry
}

func NewTaskHandler(processor *TemuProcessor) *TaskHandler {
    return &TaskHandler{
        processor: processor,
        logger: logger.GetGlobalLogger("task_handler").WithFields(logrus.Fields{
            logger.FieldComponent: "task_handler",
            logger.FieldPlatform:  "temu",
        }),
    }
}

func (h *TaskHandler) ProcessTask(ctx context.Context, task model.Task) error {
    h.logger.WithFields(logrus.Fields{
        logger.FieldTaskID:    task.ID,
        logger.FieldProductID: task.ProductID,
    }).Info("开始处理任务")
    // ...
}
```

### 示例 2: 移除 fmt.Printf

**迁移前**：
```go
// internal/crawler/shared/browser/random_config.go
func PrintConfigSummary(config *BrowserConfig) {
    fmt.Printf("=== 浏览器配置摘要 ===\n")
    fmt.Printf("平台: %s %s\n", config.FingerprintPlatform, config.FingerprintPlatformVersion)
    fmt.Printf("浏览器: %s %s\n", config.FingerprintBrand, config.FingerprintBrandVersion)
}
```

**迁移后**：
```go
import "task-processor/internal/core/logger"

func PrintConfigSummary(config *BrowserConfig) {
    log := logger.GetGlobalLogger("browser_config")
    log.WithFields(logrus.Fields{
        "platform":         config.FingerprintPlatform,
        "platform_version": config.FingerprintPlatformVersion,
        "browser":          config.FingerprintBrand,
        "browser_version":  config.FingerprintBrandVersion,
    }).Info("浏览器配置摘要")
}
```

### 示例 3: 测试代码

**迁移前**：
```go
func TestMyFunction(t *testing.T) {
    logger := logrus.New()
    logger.SetLevel(logrus.InfoLevel)
    
    // 测试代码
}
```

**迁移后**：
```go
import "task-processor/internal/core/logger"

func TestMyFunction(t *testing.T) {
    // 设置测试日志级别
    logger.SetGlobalLogLevel("debug")
    
    log := logger.GetGlobalLogger("test")
    
    // 测试代码
}
```

## 批量迁移脚本

可以使用以下命令辅助迁移：

```bash
# 查找所有使用 logrus 全局方法的地方
grep -r "logrus\.(Info\|Error\|Warn\|Debug\|Fatal)" --include="*.go" .

# 查找所有使用 fmt.Printf 的地方
grep -r "fmt\.Printf\|fmt\.Println" --include="*.go" .

# 查找所有字段命名不规范的地方
grep -r "\"taskId\"\|\"TaskID\"\|\"productId\"" --include="*.go" .
```

## 迁移检查清单

完成迁移后，检查以下项目：

- [ ] 所有文件都导入了 `task-processor/internal/core/logger`
- [ ] 移除了所有 `logrus.Info()` 等全局调用
- [ ] 移除了所有 `fmt.Printf()` 调试输出
- [ ] 所有结构体通过依赖注入获取 logger
- [ ] 字段命名统一使用 `logger.Field*` 常量
- [ ] 错误日志使用 `WithError()` 方法
- [ ] 关键操作添加了 trace_id 和 request_id
- [ ] 进度日志使用了采样（避免过多日志）
- [ ] 测试代码使用了统一的日志配置

## 常见问题

### Q: 迁移后日志格式变了？
A: 检查配置文件中的 `format` 字段，确保设置为 `json` 或 `text`。

### Q: 日志文件没有轮转？
A: 确保配置了 `max_size`、`max_backups` 和 `max_age` 参数。

### Q: 如何在不重启的情况下调整日志级别？
A: 使用 `logger.SetGlobalLogLevel("debug")` 动态调整。

### Q: 迁移后性能下降？
A: 检查是否有过多的字符串格式化操作，使用字段而不是格式化字符串。
