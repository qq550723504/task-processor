# 销售属性调试数据保存功能说明

## 功能概述

当 GPT API 调用失败或响应被截断时，系统会自动保存调试数据到文件，方便排查问题。

## 保存位置

调试数据保存在：`logs/debug/sale_attribute/` 目录下

## 文件命名规则

- 失败情况：`debug_failure_[任务ID]_[产品ID]_[时间戳].json`
- 截断情况：`debug_truncated_[任务ID]_[产品ID]_[时间戳].json`

## 保存的数据内容

```json
{
  "timestamp": "2025-12-27T22:35:14Z",
  "task_id": "4380257",
  "product_id": "B0ABCD1234",
  "system_prompt": "你是电商产品变体与销售属性生成专家...",
  "user_prompt": "基于以下产品信息生成销售属性...",
  "response": "GPT返回的响应内容（如果有）",
  "error": "具体的错误信息",
  "tokens_used": 1500,
  "model": "gpt-4",
  "is_truncated": true
}
```

## 触发条件

系统会在以下情况自动保存调试数据：

1. **API 调用失败**：网络错误、认证失败等
2. **响应为空**：GPT 返回空响应
3. **响应被截断**：达到 token 限制导致响应不完整
4. **JSON 解析失败**：无法解析 GPT 返回的 JSON 数据

## 使用方式

无需手动配置，系统会自动在失败时保存调试数据。开发人员可以通过查看保存的文件来：

1. 检查发送给 GPT 的提示词是否正确
2. 分析 GPT 返回的响应内容
3. 了解具体的错误原因
4. 优化提示词或调整 token 限制

## 日志输出

保存调试数据时会输出日志：
```
💾 调试数据已保存: logs/debug/sale_attribute/debug_failure_4380257_B0ABCD1234_20251227_223514.json
```

## 注意事项

- 调试文件可能包含敏感的产品信息，请妥善保管
- 建议定期清理旧的调试文件，避免占用过多磁盘空间
- 调试数据仅在失败或异常情况下保存，正常成功的请求不会保存
