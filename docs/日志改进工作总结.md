### 2. 核心功能实现

#### 2.1 增强的日志管理器
**文件**: `internal/core/logger/manager.go`

**功能**:
- 统一的配置管理（LogConfig）
- 支持多输出目标（控制台 + 文件）
- 动态调整日志级别
- 支持调用者信息记录
- 优雅关闭和资源清理

**配置选项**:
```yaml
log:
  level: info              # 日志级别
  format: json             # 日志格式
  output_file: logs/app.log
  max_size: 100            # 单文件最大大小(MB)
  max_backups: 10          # 保留备份数
  max_age: 30              # 保留天数
  compress: true           # 是否压缩
  console: true            # 控制台输出
  report_caller: false     # 记录调用位置
```

#### 2.2 日志轮转功能
**文件**: `internal/core/logger/rotating_writer.go`

**功能**:
- 基于文件大小的自动轮转
- 自动压缩旧日志文件（gzip）
- 按数量和时间清理旧文件
- 异步压缩和清理
- 线程安全

**轮转策略**:
- 文件达到 max_size 时自动轮转
- 保留最近 max_backups 个备份
- 删除超过 max_age 天的文件
- 文件命名: `app.log.20240304-150405.gz`

#### 2.3 Context 集成
**文件**: `internal/core/logger/context.go`

**功能**:
- 在 Context 中传递 logger
- 自动传递 trace_id 和 request_id
- 支持链式添加字段
- 从 Context 获取 logger

**使用示例**:
```go
ctx = logger.WithLogger(ctx, log)
ctx = logger.WithTraceID(ctx, "trace-123")
log := logger.FromContext(ctx, "component")
```

#### 2.4 日志辅助工具
**文件**: `internal/core/logger/helpers.go`

**提供的工具**:
- `LogOperation`: 自动记录操作耗时
- `LogProgress`: 记录进度日志
- `LogRetry`: 记录重试日志
- `LogTaskStart/Complete/Failed`: 任务生命周期
- `LogAPICall`: API调用日志
- `LogStateChange`: 状态变更日志
- `LogMetric`: 指标记录

**标准字段常量**:
```go
FieldComponent, FieldPlatform, FieldTaskID, FieldProductID,
FieldTenantID, FieldStoreID, FieldTraceID, FieldRequestID,
FieldDurationMs, FieldRetryCount, FieldErrorCode, FieldErrorType,
FieldOperation, FieldStatus
```
