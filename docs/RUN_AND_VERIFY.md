## 🔍 问题排查

### 问题1：队列使用率仍然很高（>80%）

**可能原因**: 任务处理时间超过60秒

**解决方案**:
```yaml
# 修改 config/config-temu-dev.yaml
worker:
  taskInterval: 120        # 增加到120秒
  maxFetchPerCycle: 3      # 减少到3个
```

**重启服务验证**

---

### 问题2：偶尔还是出现队列满

**可能原因**: 队列阈值不够保守

**解决方案**:
```yaml
# 修改 config/config-temu-dev.yaml
worker:
  queueThreshold: 60       # 降低到60%
```

**重启服务验证**

---

### 问题3：吞吐量太低，任务积压

**可能原因**: 获取太保守

**解决方案**:
1. 先优化任务处理速度（见下文）
2. 再适当增加获取频率

**不要直接增加获取频率！**

---

## 🎯 优化建议

### 短期优化（提升处理速度）

**目标**: 将任务处理时间从60秒降低到30-40秒

**方法**:
1. **并行化图片下载**（预计节省10-15秒）
2. **优化AI调用**（缓存、批量处理）
3. **优化Amazon爬虫**（使用缓存）

**实施后调整配置**:
```yaml
# 如果任务处理时间降低到40秒
worker:
  taskInterval: 60         # 从90秒降低到60秒
  maxFetchPerCycle: 5-8    # 可以适当增加
```

### 收集关键日志

**命令**（Windows PowerShell）:
```powershell

# 搜索队列满错误
Select-String -Path logs/app.log -Pattern "队列已满" | Out-File logs/queue_full_errors.log

Select-String -Path logs/app.log -Pattern "队列状态" | Out-File logs/queue_stats.log

# 搜索任务分发结果
```
