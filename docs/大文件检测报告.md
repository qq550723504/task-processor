## 预防措施

### 代码审查规则

1. **文件大小限制**
   - 警告：>300行
   - 拒绝：>500行（除非有充分理由）

2. **函数复杂度限制**
   - 警告：复杂度>10
   - 拒绝：复杂度>15

3. **单一职责原则**
   - 每个文件应该只有一个主要职责
   - 每个函数应该只做一件事

### 自动化检测

```bash
# 在CI/CD中添加检测
golangci-lint run --enable gocyclo --gocyclo-min-complexity=15

# 检测大文件
find . -name "*.go" -exec wc -l {} \; | awk '$1 > 300 {print $0}'
```

## 附录：检测工具使用

### gocyclo安装和使用

```bash
# 安装
go install github.com/fzipp/gocyclo/cmd/gocyclo@latest

# 检测复杂度>15的函数
gocyclo -over 15 ./...

# 显示Top 20最复杂的函数
gocyclo -top 20 ./...

# 显示平均复杂度
gocyclo -avg ./path/to/file.go
```

### 文件大小检测

```powershell
# PowerShell - 查找大文件
Get-ChildItem -Path . -Recurse -File -Include "*.go" | 
  Select-Object FullName, @{Name="Lines";Expression={(Get-Content $_.FullName | Measure-Object -Line).Lines}} | 
  Where-Object {$_.Lines -gt 300} | 
  Sort-Object Lines -Descending
```
