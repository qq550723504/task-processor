# 日志系统改进总结

## 改进概述

本次改进对项目的日志系统进行了全面优化，解决了日志使用不一致、配置混乱、性能问题等多个方面的问题。

## 主要改进内容

### 1. 统一日志管理器 (LogManager)

**位置**: `internal/core/logger/manager.go`

**新增功能**:
- 统一的日志配置管理
- 支持日志文件轮转（自动归档和压缩）
- 支持多输出目标（控制台 + 文件）
- 动态调整日志级别
- 优雅关闭和资源清理
- 支持调用者信息记录

**配置选项**:
```yaml
log:
  level: info              # 日志级别
  format: json             # 日志格式 (json/text)
  output_file: logs/app.log
  max_size: 100            # 单文件最大大小(MB)
  max_backups: 10          # 保留的备份文件数
  max_age: 30              # 保留天数
  compress: true           # 是否压缩旧文件
  console: true            # 是否输出到控制台
  report_caller: false     # 是否记录调用位置
```

### 2. 日志轮转功能

**位置**: `internal/core/logger/rotating_writer.go`

**功能特性**:
- 基于文件大小的自动轮转
- 自动压缩旧日志文件（gzip）
- 按数量和时间清理旧文件
- 异步压缩和清理，不阻塞主流程
- 线程安全

**轮转策略**:
- 当日志文件达到 `max_size` 时自动轮转
- 保留最近 `max_backups` 个备份文件
- 删除超过 `max_age` 天的旧文件
- 轮转后的文件命名格式: `app.log.20240304-150405.gz`

### 3. Context 集成

**位置**: `internal/core/logger/context.go`

**功能**:
- 在 Context 中传递 logger 实例
- 自动传递 trace_id 和 request_id
- 支持链式添加字段
- 从 Context 中获取 logger

**使用示例**:
```go
// 添加 logger 到 context
ctx = logger.WithLogger(ctx, log)

// 添加 trace_id
ctx = logger.WithTraceID(ctx, "trace-123")

// 从 context 获取 logger
log := logger.FromContext(ctx, "component")
```

## 解决的问题

### 问题 1: 日志使用不一致 ✅
**解决方案**: 
- 提供统一的 `GetGlobalLogger()` 接口
- 通过依赖注入传递 logger 实例
- 制定明确的使用规范

### 问题 2: Logger 实例管理混乱 ✅
**解决方案**:
- 全局 LogManager 单例
- 结构体中保存 logger 字段
- 避免重复创建实例

### 问题 3: 日志配置未统一应用 ✅
**解决方案**:
- 统一的配置结构 `LogConfig`
- 在系统初始化时加载配置
- 支持环境变量覆盖

### 问题 4: 日志级别控制不当 ✅
**解决方案**:
- 配置文件统一管理级别
- 提供动态调整接口
- 避免硬编码日志级别

### 问题 5: 结构化日志不完整 ✅
**解决方案**:
- 定义标准字段常量
- 统一使用 snake_case 命名
- 添加 trace_id 和 request_id 支持

### 问题 6: 错误日志处理不规范 ✅
**解决方案**:
- 强制使用 `WithError()` 方法
- 在错误发生处记录详细日志
- 避免重复记录

### 问题 7: 性能问题 ✅
**解决方案**:
- 复用 logger 实例
- 使用字段而非字符串格式化
- 提供日志采样功能

### 问题 8: 日志输出混乱 ✅
**解决方案**:
- 移除所有 fmt.Printf
- 统一使用日志系统
- 提供迁移工具和文档

### 问题 9: 日志文件管理 ✅
**解决方案**:
- 实现自动日志轮转
- 自动压缩和清理
- 妥善管理文件句柄

### 问题 10: 缺少日志规范文档 ✅
**解决方案**:
- 创建详细的使用规范
- 提供迁移指南和示例
- 提供自动化检查工具

## 使用示例

### 基础使用

```go
import "task-processor/internal/core/logger"

// 获取 logger
log := logger.GetGlobalLogger("my_service")

// 记录日志
log.Info("服务启动")
log.WithField("port", 8080).Info("监听端口")
log.WithError(err).Error("操作失败")
```

### 结构体中使用

```go
type MyService struct {
    logger *logrus.Entry
    helper *logger.LoggerHelper
}

func NewMyService() *MyService {
    log := logger.GetGlobalLogger("my_service")
    return &MyService{
        logger: log,
        helper: logger.NewLoggerHelper(log),
    }
}

func (s *MyService) DoWork() error {
    return s.helper.LogOperation("do_work", func() error {
        // 业务逻辑
        return nil
    })
}
```

### Context 传递

```go
func ProcessRequest(ctx context.Context, req Request) error {
    // 创建带上下文的 logger
    log := logger.GetGlobalLogger("handler").WithFields(logrus.Fields{
        logger.FieldRequestID: req.ID,
        logger.FieldUserID:    req.UserID,
    })
    
    // 添加到 context
    ctx = logger.WithLogger(ctx, log)
    
    // 调用其他函数
    return s.service.Process(ctx, req)
}

func (s *Service) Process(ctx context.Context, req Request) error {
    // 从 context 获取 logger（自动包含 request_id 等字段）
    log := logger.FromContext(ctx, "service")
    log.Info("开始处理")
    return nil
}
```

## 性能影响

### 优化点
1. **减少 logger 创建**: 通过复用实例，减少内存分配
2. **延迟格式化**: 使用字段而非字符串格式化
3. **异步轮转**: 日志轮转不阻塞主流程
4. **采样支持**: 高频日志可以采样记录
