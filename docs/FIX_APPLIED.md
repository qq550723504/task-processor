## 📊 关键配置说明

### taskInterval: 90秒
**原因**：单个任务处理时间约60秒
- 90秒内可处理：5 × (90/60) = 7.5 个任务
- 留有充足余量，避免队列积压

### maxFetchPerCycle: 5个
**原因**：匹配并发数
- 5个任务需要60秒处理完
- 下次获取时队列已有空位
- 避免一次性填满队列

### queueThreshold: 75%
**原因**：保守的压力阈值
- 20个槽位 × 75% = 15个任务
- 留有5个槽位的安全余量
- 超过阈值时暂停获取，等待队列消化

## 🔧 问题2修复：SKU编码检查日志优化

### 问题描述
日志中频繁出现警告：`商品ID为空，跳过SKU编码检查`

### 根本原因
新建产品时 `GoodsID` 为空是正常的（只有提交后才会有），但代码使用 `Warn` 级别记录，造成日志噪音。

### 修复内容
**文件**: `platforms/temu/handlers/out_goods_sn_check_handler.go`

```go
// 修改前
if ctx.TemuProduct.GoodsBasic.GoodsID == "" {
    h.logger.Warn("商品ID为空，跳过SKU编码检查")
    return nil
}

// 修改后
if ctx.TemuProduct.GoodsBasic.GoodsID == "" {
    h.logger.Debug("新建产品，GoodsID尚未生成，跳过SKU编码检查")
    return nil
}
```

### 效果
- ✅ 将日志级别从 `Warn` 改为 `Debug`
- ✅ 添加更清晰的说明（新建产品的正常情况）
- ✅ 减少日志噪音
