# 快速修复参考卡片

## 问题诊断

### 症状
- ❌ 大量任务提交失败
- ❌ 错误信息：`工作池队列已满`
- ❌ 任务等待时间过长（100+ 小时）

### 根本原因
**任务获取速度 > 任务处理速度**

```
当前状态：
- 任务处理时间：60 秒/个
- 获取间隔：30 秒
- 单次获取：20 个任务
- 工作池容量：5 并发 + 20 缓冲 = 25 个

问题：
30 秒内只能处理 2.5 个任务（5 × 30/60）
但每次获取 20 个任务，导致队列快速填满
```

---

## 快速修复（5分钟）

### 步骤1：修改配置文件

编辑 `config/config-temu-dev.yaml`：

```yaml
worker:
  concurrency: 5           # 不变
  buffer_size: 20          # 不变
  task_interval: 90        # 改：30 → 90
  max_fetch_per_cycle: 5   # 新增
  queue_threshold: 75      # 新增
```

### 步骤2：重启服务

```bash
# 停止当前服务（Ctrl+C）
# 重新启动
go run ./cmd/temu-web
```

### 步骤3：验证效果

观察日志，应该看到：
```
✅ 每90秒获取一次任务
✅ 每次获取 ≤ 5 个任务
✅ 队列使用率保持在 75% 以下
✅ 没有"队列已满"错误
```

## 配置说明

### 核心参数

| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `task_interval` | 30秒 | 90秒 | 获取间隔 = 任务处理时间 × 1.5 |
| `max_fetch_per_cycle` | 无限制 | 5个 | 单次获取 ≤ 并发数 |
| `queue_threshold` | 无 | 75% | 队列使用率阈值 |

### 计算逻辑

```
任务处理时间 = 60 秒
并发数 = 5

理论吞吐量 = 5 / 60 = 0.083 任务/秒 = 5 任务/分钟

安全获取间隔 = 60 × 1.5 = 90 秒
安全获取数量 = 5 个（匹配并发数）

90秒内可处理 = 5 × (90/60) = 7.5 个任务
获取 5 个任务，留有 2.5 个任务的余量 ✅
```

## 故障排查

### 如果修复后仍有问题

#### 问题1：队列使用率仍然很高（>80%）
**原因**：任务处理时间可能超过 60 秒

**解决**：
```yaml
task_interval: 120  # 增加到 120 秒
max_fetch_per_cycle: 3  # 减少到 3 个
```

#### 问题2：吞吐量太低，任务积压
**原因**：获取太保守

**解决**：
1. 先优化任务处理速度（见下文）
2. 再适当增加获取频率

#### 问题3：偶尔还是出现队列满
**原因**：队列阈值设置不够保守

**解决**：
```yaml
queue_threshold: 60  # 降低到 60%
```
