# ListingKit 付费试点产品目录与用量政策

## 套餐与准入

唯一首发套餐代码为 `paid_pilot`，仅限邀请制。它不是公开注册或自助 SaaS 套餐：platform-admin 必须根据已批准的合同或订单人工开通、暂停和管理到期；没有自助购买、自动升级或自动收费。

每次人工开通、套餐变更、配额调整、暂停和到期处理都必须记录 tenant、actor、操作前后值、原因、时间和合同/订单引用。价格、币种、账期和具体配额以该租户已批准的合同/订单和 platform-admin 台账为准，不得由内部用量推断，也不得向其他租户暴露。

## 能力矩阵

| 能力 | `paid_pilot` 默认状态 | 额外门禁 |
| --- | --- | --- |
| 创建 ListingKit 任务 | 可用 | 已完成租户准入，以及现有身份、授权和运行时配额控制 |
| SDS 来源 | 可用 | 已配置来源/店铺并通过适用的 preflight |
| Studio 设计生成 | 可用 | 已完成租户准入；仅成功任务提交用量 |
| 商品图生成 | 可用 | 已完成租户准入；仅成功任务提交用量 |
| SHEIN 保存草稿 | 可用 | 已配置店铺、preflight 和现有提交安全控制 |
| SHEIN 正式发布 | 默认关闭 | 独立 `shein_publish` entitlement、该租户 preflight、草稿验收和显式业务批准 |
| 1688 来源 | 保持关闭 | M2 来源追溯与安全门禁完成后的独立政策更新 |

将 `paid_pilot` 分配给租户本身不会开启正式发布。关闭 `shein_publish` 后，既有任务仍可查看，但不能开始新的发布动作。1688 在保持关闭期间不能因套餐分配、内部重试或前端隐藏/显示变化而绕过该门禁。

## 用量和收费语义

首发使用下列稳定指标：

| 指标 | 首发语义 |
| --- | --- |
| `listing_tasks_created` | 运营容量和准入配额指标；首发不作为独立计费完成项。 |
| `studio_design_jobs_succeeded` | Studio 设计任务成功完成后提交一次用量。 |
| `product_image_jobs_succeeded` | 商品图任务成功完成后提交一次用量。 |
| `shein_drafts_succeeded` | SHEIN 远端草稿成功后提交一次用量。 |
| `shein_publishes_succeeded` | 已启用 `shein_publish` 且远端正式发布成功后提交一次用量。 |
| `storage_bytes_current` | 租户当前保留对象的字节占用，不按上传量或 byte-hours 计算。 |

失败、取消、平台拒绝和工程重放不计费。重试或恢复必须复用同一业务事件，不能创建额外客户用量；远端结果未知时不得进入计费，必须先完成 reconciliation。删除对象会降低后续 `storage_bytes_current`，但不会为已成功的生成或远端提交制造退款事件。

AI 的内部成本必须被记录用于成本核算，但首发不按单次内部 AI 调用向客户收费。客户可见的订阅/配额信息只能包含自己的套餐、已启用能力、指标、已用量、限额和支持入口；不得显示合同价格、其他租户数据或内部 AI 成本。

## 试点运营边界

套餐到期或被暂停后，系统必须按已记录的政策拒绝新的受限动作，同时保留历史记录和审计信息。平台管理员不得通过内部 retry 或 recovery 绕过 entitlement 或重复计量。

本政策不启用任何收费。只有 PAY-041 的幂等 usage ledger、PAY-042 的统一入口执行、PAY-043 的人工商业台账和 PAY-044 的对账/补记能力完成并通过各自验收后，才可以根据已批准合同向试点租户收费。
