# Task Processor 调用链分析文档

## 📋 概述

本文档详细分析了 Task Processor 系统从 main 入口开始的完整调用链路，包括系统初始化、组件启动、任务处理和优雅关闭的全过程。

## 🚀 系统启动流程

### 1. Main 入口 (`cmd/task/main.go`)

```
main()
├── monitoring.RecordProcessStartTime()          // 记录进程启动时间
├── utils.SetupLogger()                          // 设置日志系统
├── container.NewContainer(logger)               // 创建依赖注入容器
└── runApplication(container)                    // 运行应用主逻辑
```

### 2. 应用运行流程 (`runApplication`)

```
runApplication()
├── utils.PrintVersionInfo()                     // 显示版本信息
├── container.Initialize()                       // 初始化容器
├── container.LoadConfig("")                     // 加载配置
├── container.InitializeAuth()                   // 初始化认证
├── context.WithCancel()                         // 创建应用上下文
├── container.StartAll(ctx, appVersion)          // 启动所有组件
├── waitForShutdown()                           // 等待关闭信号
└── gracefulShutdown()                          // 优雅关闭
```

## 🏗️ 容器初始化流程

### 1. 容器创建 (`internal/infra/container/container.go`)

```
Container.Initialize()
├── initializeServices()                         // 初始化基础服务
│   ├── service.NewConfigService()              // 配置服务
│   ├── service.NewAuthService()                // 认证服务
│   ├── service.NewUpdaterService()             // 更新服务
│   └── service.NewProcessorService()           // 处理器服务
└── initialized = true                          // 标记初始化完成
```

### 2. 配置加载 (`LoadConfig`)

```
LoadConfig()
├── configService.LoadConfig(configPath)        // 加载配置文件
├── cfg.ValidateAndLog(logger)                  // 验证配置
└── c.config = cfg                              // 保存配置
```

### 3. 认证初始化 (`InitializeAuth`)

```
InitializeAuth()
├── authService.InitializeClientCredentials()   // 初始化客户端凭证
└── c.authClient = authClient                   // 保存认证客户端
```

## ⚙️ 组件启动流程

### 1. 启动所有组件 (`StartAll`)

```
StartAll()
├── updaterService.StartAutoUpdater()           // 启动自动更新器
└── processorService.StartProcessors()          // 启动处理器服务
```

### 2. 处理器服务启动 (`StartProcessors`)

```
StartProcessors() [processor_lifecycle.go]
├── initializeSharedResources()                 // 初始化共享资源
├── startProcessors()                           // 启动平台处理器
├── startTaskFetcher()                          // 启动任务获取器
├── startSchedulerService()                     // 启动调度服务
├── initializeMonitoring()                      // 初始化监控组件
├── lifecycleManager.StartAll()                 // 启动生命周期管理的组件
└── startStatusMonitor()                        // 启动状态监控
```

### 3. 平台处理器启动 (`startProcessors`)

```
startProcessors() [processor_manager.go]
├── if cfg.Platforms.Temu.Enabled:
│   └── startTemuProcessor()                    // 启动TEMU处理器
│       ├── GetSharedAmazonProcessor()          // 获取共享Amazon处理器
│       ├── temu.NewTemuProcessor()             // 创建TEMU处理器
│       └── temuProcessor.Start()               // 启动处理器
└── if cfg.Platforms.Shein.Enabled:
    └── startSheinProcessor()                   // 启动SHEIN处理器
        ├── GetSharedAmazonProcessor()          // 获取共享Amazon处理器
        ├── pipeline.NewSheinProcessor()        // 创建SHEIN处理器
        └── sheinProcessor.Start()              // 启动处理器
```

### 4. 任务获取器启动 (`startTaskFetcher`)

```
startTaskFetcher()
├── 收集所有平台的任务提交器
│   ├── NewTaskSubmitterAdapter(temuProcessor)  // TEMU任务提交器适配器
│   └── NewTaskSubmitterAdapter(sheinProcessor) // SHEIN任务提交器适配器
├── task.NewUnifiedTaskFetcher()                // 创建统一任务获取器
└── lifecycleManager.Register()                 // 注册到生命周期管理器
```

### 5. 调度服务启动 (`startSchedulerService`)

```
startSchedulerService() [scheduler_service.go]
├── SetGlobalConfig(cfg)                        // 设置全局配置
├── NewSchedulerService(logger)                 // 创建调度服务
└── schedulerService.Start(ctx)                 // 启动调度服务
    ├── initializeResources()                   // 初始化资源
    │   └── GetSharedManagementClientInstance() // 获取共享管理客户端
    └── startScheduledTasks()                   // 启动调度任务
        ├── scheduler.NewManager()              // 创建调度管理器
        ├── getPlatformConfigs()                // 获取平台配置
        └── startPlatformTasks()                // 启动平台任务
```

### 6. 监控组件初始化 (`initializeMonitoring`)

```
initializeMonitoring()
├── monitoring.NewMetricsCollector()            // 创建指标收集器
├── monitoring.NewHealthChecker()               // 创建健康检查器
├── registerHealthChecks()                      // 注册健康检查
│   ├── ConfigHealthCheck                       // 配置健康检查
│   ├── ManagementClientHealthCheck             // 管理客户端健康检查
│   └── ProcessorHealthCheck                    // 处理器健康检查
├── lifecycleManager.Register()                 // 注册到生命周期管理器
└── collectBusinessMetrics()                    // 启动业务指标收集
```

## 🔄 运行时处理流程

### 1. 任务处理流程

```
任务处理流程
├── TaskFetcher.FetchTasks()                    // 获取任务
├── TaskSubmitter.SubmitTask()                  // 提交任务到处理器
├── Processor.ProcessTask()                     // 处理器处理任务
│   ├── WorkerPool.Submit()                     // 提交到工作池
│   ├── Worker.Execute()                        // 工作者执行任务
│   └── Pipeline.Process()                      // 管道处理
└── TaskReporter.ReportResult()                 // 报告处理结果
```

### 2. 调度任务流程

```
调度任务流程
├── SchedulerManager.Schedule()                 // 调度任务
├── TaskExecutor.Execute()                      // 执行任务
│   ├── 核价任务 (PricingTask)
│   ├── 产品同步任务 (ProductSyncTask)
│   ├── 库存同步任务 (InventorySyncTask)
│   └── 活动报名任务 (ActivityRegistrationTask)
└── TaskMonitor.Monitor()                       // 监控任务执行
```

### 3. 监控和健康检查

```
监控流程
├── MetricsCollector.Collect()                  // 收集指标
│   ├── 队列大小 (queue_size)
│   ├── 队列使用率 (queue_usage_percent)
│   └── 处理器运行状态 (processor_running)
├── HealthChecker.Check()                       // 健康检查
│   ├── 配置检查
│   ├── 管理客户端检查
│   └── 处理器检查
└── StatusMonitor.Monitor()                     // 状态监控
```

## 🛑 系统关闭流程

### 1. 信号处理 (`waitForShutdown`)

```
waitForShutdown()
├── signal.Notify()                             // 监听系统信号
├── <-sigChan                                   // 等待信号
└── cancel()                                    // 取消上下文，通知所有组件停止
```

### 2. 优雅关闭 (`gracefulShutdown`)

```
gracefulShutdown()
├── context.WithTimeout(30s)                   // 设置关闭超时
└── container.StopAll(ctx)                     // 停止所有组件
    ├── processorService.StopProcessors()       // 停止处理器服务
    └── closeSharedResources()                  // 关闭共享资源
```

### 3. 处理器服务停止 (`StopProcessors`)

```
StopProcessors()
├── cancel()                                    // 取消上下文
├── lifecycleManager.StopAll()                  // 停止生命周期管理的组件
├── stopAllProcessors()                         // 停止所有处理器
│   ├── temuProcessor.Close()                   // 关闭TEMU处理器
│   ├── sheinProcessor.Close()                  // 关闭SHEIN处理器
│   ├── CloseSharedAmazonProcessor()            // 关闭共享Amazon处理器
│   └── CloseSharedManagementClient()           // 关闭共享管理客户端
└── schedulerService.Stop()                     // 停止调度服务
```

## 🏗️ 架构设计特点

### 1. 依赖注入容器
- **统一管理**: 所有组件通过容器统一创建和管理
- **生命周期控制**: 容器负责组件的启动和停止
- **资源共享**: 通过容器实现资源的共享和复用

### 2. 生命周期管理
- **统一接口**: 所有组件实现统一的生命周期接口
- **优雅启停**: 支持组件的优雅启动和停止
- **错误处理**: 完善的错误处理和恢复机制

### 3. 模块化设计
- **职责分离**: 每个模块职责单一，边界清晰
- **松耦合**: 模块间通过接口交互，降低耦合度
- **可扩展**: 支持新平台和功能的快速扩展

### 4. 监控和健康检查
- **全面监控**: 覆盖系统各个层面的监控
- **健康检查**: 实时检查组件健康状态
- **指标收集**: 收集关键业务和系统指标

## 📊 关键组件说明

### 1. 核心组件
- **Container**: 依赖注入容器，管理所有组件
- **ProcessorService**: 处理器服务，管理平台处理器
- **SchedulerService**: 调度服务，管理周期性任务
- **TaskFetcher**: 任务获取器，从管理系统获取任务

### 2. 平台处理器
- **TemuProcessor**: TEMU平台处理器
- **SheinProcessor**: SHEIN平台处理器
- **AmazonProcessor**: Amazon处理器（共享）

### 3. 监控组件
- **MetricsCollector**: 指标收集器
- **HealthChecker**: 健康检查器
- **StatusMonitor**: 状态监控器

### 4. 基础设施
- **Config**: 配置管理
- **Logger**: 日志系统
- **Auth**: 认证系统
- **Management**: 管理系统客户端

### 3. 扩展能力

3. **强大的扩展能力**: 支持多平台和功能的快速扩展
