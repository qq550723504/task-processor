# 自动更新器循环问题修复

## 问题描述
程序启动后检测到版本是 1.0.0，然后一直下载、更新、重启，陷入无限循环。

## 根本原因
1. **启动时立即检查更新**：程序启动后立即执行更新检查
2. **没有更新冷却期**：刚更新完的程序重启后又立即检查更新
3. **版本比较不准确**：简单的字符串比较无法正确处理语义化版本号
4. **缺少更新标记**：没有记录已更新的版本，导致重复更新

## 修复方案

### 1. 添加更新冷却期
- 程序启动时检查 `.update-marker` 文件的修改时间
- 如果1小时内更新过，跳过启动时的更新检查
- 延迟30秒后再进行首次更新检查

### 2. 改进版本比较
- 实现语义化版本号比较（支持 1.0.0, 1.0.1, 2.0.0 等格式）
- 支持 'v' 前缀（如 v1.0.0）
- 正确处理版本号大小关系

### 3. 更新标记机制
- 下载更新成功后，在重启**之前**创建 `.update-marker` 文件
- 文件内容包含版本号和更新时间
- 启动时检查此文件，避免重复更新同一版本

### 4. 增强日志输出
- 显示当前版本和远程版本
- 记录版本比较结果
- 显示更新冷却期状态

## 修复的代码文件
- `updater/updater.go`

## 关键改动

### Start() 函数
```go
// 检查是否刚刚更新过（避免更新循环）
if u.isRecentlyUpdated() {
    logrus.Info("检测到最近刚更新过，跳过启动时的更新检查")
} else {
    // 延迟30秒后再检查（给程序启动留出时间）
    logrus.Info("将在30秒后进行首次更新检查...")
    time.Sleep(30 * time.Second)
    u.checkAndUpdate()
}
```

### checkAndUpdate() 函数
```go
// 比较版本 - 使用语义化版本比较
cmp := compareVersion(latestVersion.Version, u.currentVersion)
if cmp <= 0 {
    logrus.Infof("当前已是最新版本 (本地: %s, 远程: %s)", u.currentVersion, latestVersion.Version)
    return
}

// 检查是否已经更新过这个版本（防止重复更新）
if u.isAlreadyUpdated(latestVersion.Version) {
    logrus.Infof("版本 %s 已经更新过，跳过", latestVersion.Version)
    return
}
```

### 新增函数
- `compareVersion()`: 语义化版本号比较
- `splitVersion()`: 分割版本号为数字数组
- `isRecentlyUpdated()`: 检查是否最近1小时内更新过
- `markAsUpdated()`: 标记已更新版本（在重启前调用）

## 使用说明

### 正常使用
程序会自动处理更新循环问题，无需手动干预。

### 手动清理（如果需要）
如果需要强制重新检查更新，删除 `.update-marker` 文件：
```bash
del .update-marker
```

### 禁用自动更新
在配置文件中设置：
```yaml
updater:
  enabled: false
```

## 测试验证
1. 启动程序，观察日志输出
2. 如果检测到最近更新过，会显示："检测到最近刚更新过，跳过启动时的更新检查"
3. 如果没有更新过，会显示："将在30秒后进行首次更新检查..."
4. 版本比较会显示："当前已是最新版本 (本地: 1.0.0, 远程: 1.0.0)"

## 预防措施
- 更新冷却期：1小时内不会重复检查
- 启动延迟：30秒后才进行首次检查
- 版本标记：记录已更新的版本号
- 智能比较：正确处理版本号大小关系
