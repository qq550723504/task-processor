# 日志系统快速参考

## 🚀 快速开始

### 基础使用
```go
import "task-processor/internal/core/logger"

// 获取logger
log := logger.GetGlobalLogger("my_service")

// 记录日志
log.Info("服务启动")
log.WithField("port", 8080).Info("监听端口")
log.WithError(err).Error("操作失败")
```

### 在结构体中使用
```go
type MyService struct {
    logger *logrus.Entry
}

func NewMyService() *MyService {
    return &MyService{
        logger: logger.GetGlobalLogger("my_service"),
    }
}
```

## 🛠️ 辅助工具

### LoggerHelper
```go
helper := logger.NewLoggerHelper(log)

// 自动记录操作耗时
helper.LogOperation("fetch_data", func() error {
    return fetchData()
})

// 记录进度
helper.LogProgress(50, 100, "处理中")

// 记录任务
helper.LogTaskStart(taskID, productID)
helper.LogTaskComplete(taskID, duration)
helper.LogTaskFailed(taskID, err)
```

## 📋 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| Debug | 详细调试信息 | 函数参数、中间结果 |
| Info | 重要业务流程 | 任务开始/完成、状态变更 |
| Warn | 警告信息 | 重试操作、使用默认值 |
| Error | 错误信息 | API失败、数据验证失败 |
| Fatal | 致命错误 | 配置缺失、服务无法启动 |

## ✅ 最佳实践

### DO ✅
```go
// 使用标准字段常量
log.WithField(logger.FieldTaskID, taskID)

// 错误使用 WithError
log.WithError(err).Error("操作失败")

// 使用结构化字段
log.WithFields(logrus.Fields{
    logger.FieldTaskID: taskID,
    "priority": priority,
}).Info("任务处理")

// 复用logger实例
type Service struct {
    logger *logrus.Entry
}
```

### DON'T ❌
```go
// ❌ 不要使用全局方法
logrus.Info("message")

// ❌ 不要使用 fmt.Printf
fmt.Printf("debug: %v\n", data)

// ❌ 不要重复创建logger
logger := logrus.New()

// ❌ 不要使用驼峰命名
log.WithField("taskId", taskID)

// ❌ 不要字符串格式化
log.Infof("task %d failed", taskID)
```

## 🔧 配置示例

```yaml
log:
  level: info              # debug, info, warn, error
  format: json             # json, text
  output_file: logs/app.log
  max_size: 100            # MB
  max_backups: 10          # 保留备份数
  max_age: 30              # 保留天数
  compress: true           # 是否压缩
  console: true            # 控制台输出
```

## 🔍 Context 使用

```go
// 添加logger到context
ctx = logger.WithLogger(ctx, log)
ctx = logger.WithTraceID(ctx, "trace-123")

// 从context获取logger
log := logger.FromContext(ctx, "component")
```

## 📊 常用模式

### 任务处理
```go
log.WithFields(logrus.Fields{
    logger.FieldTaskID: taskID,
    logger.FieldProductID: productID,
}).Info("开始处理任务")

// 处理逻辑...

log.WithFields(logrus.Fields{
    logger.FieldTaskID: taskID,
    logger.FieldDurationMs: duration.Milliseconds(),
}).Info("任务处理完成")
```

### 错误处理
```go
if err != nil {
    log.WithError(err).WithFields(logrus.Fields{
        logger.FieldOperation: "fetch_data",
        "url": apiURL,
    }).Error("操作失败")
    return fmt.Errorf("获取数据失败: %w", err)
}
```

### 进度日志
```go
for i, item := range items {
    if (i+1)%100 == 0 || i == len(items)-1 {
        helper.LogProgress(i+1, len(items), "批处理进度")
    }
    processItem(item)
}
```

### API调用
```go
start := time.Now()
resp, err := client.Get(url)
duration := time.Since(start)

helper.LogAPICall("GET", url, resp.StatusCode, duration)
```

## 🆘 常见问题

**Q: 如何动态调整日志级别？**
```go
logger.SetGlobalLogLevel("debug")
```

**Q: 如何在测试中使用？**
```go
func TestMyFunction(t *testing.T) {
    logger.SetGlobalLogLevel("debug")
    log := logger.GetGlobalLogger("test")
    // 测试代码
}
```

**Q: 日志文件在哪里？**
默认在 `logs/` 目录，可通过配置修改。

**Q: 如何查看压缩的日志？**
```bash
zless logs/app.log.20240304-150405.gz
```
