# TEMU 智能核价系统

## 概述

智能核价系统根据店铺或类目的利润率规则，自动对待核价商品做出决策（接受、拒绝或重新报价）。

## 核心逻辑

### 1. 获取待核价列表

系统定期从 TEMU 平台获取待核价商品列表，包含：
- 商品基本信息（ID、名称、SKU等）
- 当前成本价（供应商价格）
- 平台推荐价格
- 类目信息

### 2. 获取上架记录映射

从管理系统获取商品的上架记录映射，包含：
- `costPrice`: 原始成本价（上架时的成本）
- `salePriceMultiplier`: 售价倍数（如果没有成本价，用此倍数反推）
- `profitRuleId`: 利润规则ID
- 其他上架时的配置信息

### 3. 计算原始成本价

根据上架记录映射计算原始成本价：
- 如果有 `costPrice`，直接使用
- 如果没有 `costPrice` 但有 `salePriceMultiplier`，则：
  ```
  原始成本价 = 当前价格 / 售价倍数
  ```

### 4. 计算利润率

利润率计算公式：
```
利润率 = (平台推荐价 - 原始成本价) / 平台推荐价 × 100%
```

### 5. 决策逻辑

根据平台推荐价与原始成本价对比：

#### 情况 1: 平台推荐价 >= 原始成本价
- **决策**: 接受平台报价
- **原因**: 不会亏损，可以接受

#### 情况 2: 平台推荐价 < 原始成本价
- **决策**: 拒绝报价
- **原因**: 会亏损，不能接受

#### 特殊情况: 数据不完整
- **决策**: 跳过
- **原因**: 价格数据缺失或上架记录获取失败

### 6. 执行决策

根据决策结果调用相应的 TEMU API：
- **接受**: 调用 `AcceptPrice` API
- **拒绝**: 调用 `RejectPrice` API
- **跳过**: 不执行任何操作

## 使用方式

### 1. 确保商品已上架

商品必须先通过正常流程上架到 TEMU，系统会记录：
- 原始成本价
- 售价倍数
- 利润规则等配置信息

### 2. 启用智能核价

系统只支持智能核价模式，在 `auto_pricing_handler.go` 中自动启用：

```go
action := temu.ActionSmart
```

### 3. 调度执行

系统会按配置的时间间隔自动执行智能核价任务，处理所有待核价商品。

## 统计信息

每次执行完成后，系统会输出统计信息：
- 总处理数量
- 接受数量
- 拒绝数量
- 重新报价数量
- 跳过数量
- 成功数量
- 失败数量

## 日志记录

系统会详细记录每个商品的决策过程：
- 商品信息（名称、SKU等）
- 原始成本价和平台推荐价
- 计算出的利润率
- 决策动作和原因

## 注意事项

1. **上架记录**: 商品必须先上架才能进行智能核价，系统依赖上架记录中的成本价信息
2. **价格数据完整性**: 确保待核价列表返回完整的价格信息
3. **API 限流**: 注意 TEMU API 的调用频率限制
4. **错误处理**: 系统会跳过处理失败的商品，继续处理其他商品
5. **Cookie 管理**: 确保店铺的 Cookie 有效，否则会暂停任务

## 核心原则

智能核价的核心原则是：**不亏本**
- 只要平台推荐价 >= 原始成本价，就接受
- 如果平台推荐价 < 原始成本价，就拒绝
- 这样可以确保不会亏本销售

## 扩展性

系统设计支持未来扩展：
- 可以添加更复杂的决策规则（如目标利润率、最低利润率等）
- 可以支持重新报价功能
- 可以添加基于历史数据、竞品价格的分析
- 可以支持批量操作优化性能
