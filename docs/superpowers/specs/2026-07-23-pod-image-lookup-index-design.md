# POD 原图检索索引设计

## 目标

将 SHEIN POD 原图查询从读取最多 5,000 条完整任务 JSON 后在应用内筛选，改为从一张轻量、可索引的检索表直接查询。该接口必须保留现有返回结构、店铺隔离和 SKU/SPU/版本/任务 ID 等检索能力，并在 UI 代理的显式 60 秒保护性上限内响应。

## 现状与根因

`LookupSheinPODImages` 会选择任务表中的 `request`、`result` 和店铺快照三个 JSON 字段，最多读取 5,000 条，再构建记录并做规范化 contains 匹配。线上店铺 870 的请求被 Next.js 代理在 15,007ms 终止并返回 504。该路径没有专用代理超时，因此使用默认 15 秒。

## 方案

新增 `listingkit_shein_pod_image_indexes`，每个任务一行。该表只保存查询和展示需要的字段：任务、租户、用户、店铺、状态、商品与 SKU 标识、SHEIN SPU/版本、AI 原图、SDS 主图/画廊图、创建/更新时间，以及这些可查询字段规范化值的 SHA-256 固定长度检索键。

每个精确查询字段使用 `(store_id, <field>_lookup_key)` 的 B-tree 索引，其中检索键为规范化值的 64 字符 SHA-256 十六进制编码，避免 PostgreSQL B-tree 对长商品文本或 URL 索引项的大小限制。查询词沿用当前的规范化规则（忽略 `-`、`_` 和空白、转大写），计算同样的固定长度键后命中索引，再用不建索引的规范化原值做相等校验作为哈希碰撞保护；响应仍返回未哈希的原始展示值。结果按更新时间倒序、限制 50 条。这样避免 JSON 读取、反序列化和未索引 contains 谓词。页面提示的 SKU、SPU、版本和任务 ID 都是精确标识，且现有占位示例是完整 SKU；本次不支持任意子串匹配，以换取有界性能。

## 写入与回填

任务创建后、任务结果保存/变更后以及 SDS 选项替换后，在与任务写入相同的数据库事务中 upsert 或删除索引行。只有 `BuildSheinPODImageLookupRecord` 能构建出可检索记录的任务保留索引；其余任务移除已有索引行，避免过期数据。

新增可重复执行的回填函数和受控迁移脚本入口 `scripts/listingkit-shein-pod-image-index-backfill`。工具必须显式传入 `-config`，只迁移 POD 图片索引表，不执行其他 runtime migration。部署镜像包含该工具以及不自动执行的 Kubernetes Job 模板。部署顺序为：部署包含 schema 和工具的同一不可变镜像、用已部署镜像 tag 创建一次性 backfill Job、等待 Job 成功、再以已知 SKU 验证端点响应与代理日志。命令按固定批次扫描任务 ID，并在每个写事务中以 `SELECT ... FOR UPDATE` 重新读取任务后构建、upsert 或删除索引；这保证批次读取后发生的并发任务更新不会被旧快照覆盖或删除。重复运行不会产生重复数据。应用启动仅执行 schema migration，不在用户查询路径或启动路径扫描旧任务。

## API、错误处理与可观测性

HTTP 路径、参数和响应 JSON 保持不变。GET `/shein-pod-image-lookup/stores/:store_id` 使用命名的 60 秒代理上限；这是索引尚未预热、数据库短暂波动时避免默认 15 秒过早中断的保护性上限，而非性能方案，查询性能仍依赖索引表。索引表为空时返回空数组与零总数，而不是回退到全任务扫描。数据库错误继续映射为现有的 `shein_pod_image_lookup_failed`。新增结构化日志/指标字段仅记录候选数、匹配数和耗时，不记录图片 URL 或搜索词。

## 测试与验收

- SQLite 仓储测试验证 schema、创建与结果更新会同步索引。
- 查询测试验证店铺与租户隔离、SKU（含连字符规范化）、SPU、版本和任务 ID 的匹配、排序、限制和无索引回退。
- Dry-run SQL 测试断言查询只访问索引表，不选择任务 JSON，也不生成 JSON 文本谓词。
- 回填测试验证幂等、批处理、过期索引删除，以及批次读取后并发更新不会被旧快照覆盖或删除。
- UI 代理测试验证 POD 原图路径获得 60 秒的显式保护性上限；性能正确性依赖索引表而非扩大超时。

## 非目标

- 不修改已有任务 JSON 的结构。
- 不引入 PostgreSQL `pg_trgm` 扩展或 JSON GIN/表达式索引。
- 不在本次为任意关键词子串搜索提供兼容回退。
