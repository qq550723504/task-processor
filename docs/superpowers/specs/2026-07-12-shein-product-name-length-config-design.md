# SHEIN 产品名与 SKC 名动态长度配置设计

## 背景

当前 SKC 英文名称上限在 `internal/shein/product/skc/translation.go` 中硬编码为 150，产品多语言名称生成流程没有按分类、语言应用 SHEIN 的名称长度配置。这会导致服务端行为与 SHEIN 发布页的实际限制不一致。

SHEIN 提供接口：

`POST /spmp-api-prefix/spmp/product/publish/config/query_product_name_length_config`

请求体包含最终分类 ID：

```json
{"category_id": 1772}
```

响应的 `info` 数组按语言给出 `max_length`，例如英文 150、简体中文 100、繁体中文 105。

## 目标

- 产品名和 SKC 名均使用接口返回的对应语言长度上限。
- 同一发布任务仅查询一次分类名称长度配置。
- 接口不可用时不阻塞发布，记录告警并使用兼容默认行为。
- 按 Unicode 字符而非 UTF-8 字节计算与截断名称长度。

## 非目标

- 不改变产品描述长度规则。
- 不新增跨任务持久缓存。
- 不修改名称的最小长度、翻译语言集合或分类选择逻辑。
- 不重构与名称长度无关的产品生成流程。

## 方案

### API 层

在现有 SHEIN product API 中增加名称长度配置查询：

- endpoint 常量及 getter；
- 请求模型 `category_id`；
- 响应模型包含 `language` 和 `max_length`；
- `ProductAPI` 公开查询方法。

API 层负责校验 SHEIN 响应状态并返回结构化配置，不负责业务兜底。

### 任务级配置

在最终 `category_id` 已写入 `ProductData` 后查询一次配置，并将规范化后的 `map[language]maxLength` 保存到任务上下文。语言键统一转为小写并去除首尾空白，非正数上限忽略。

若请求失败，记录包含分类 ID 和错误原因的告警，并保存空配置，使后续流程使用默认行为而不重复请求。

### 名称生成

产品名生成和 SKC 名生成接收同一份任务级配置：

- 对每个 `LanguageContent` 按其语言查找上限；
- 命中配置时，在翻译或 AI 优化完成后截断；
- 英文 SKC AI 提示中的最大长度使用动态英文上限；
- 英文配置缺失时使用现有默认值 150；
- 其他语言配置缺失时保持当前行为，不新增截断。

截断函数使用 rune 计数，并尽量在接近上限的位置按空格截断英文；中文等无空格文本则严格截到字符上限。返回值去除首尾空白。

### 数据流

1. 分类选择完成，`ProductData.CategoryID` 可用。
2. 产品 API 查询该分类的名称长度配置。
3. 结果规范化后写入任务上下文；失败则写入空配置并告警。
4. 产品多语言名称生成读取配置并截断各语言结果。
5. SKC 多语言名称生成读取同一配置，动态构造英文优化约束并截断结果。
6. 现有发布流程提交处理后的产品数据。

## 错误处理

- 网络错误、非成功响应或响应解析失败：记录告警，英文回退 150，其他语言不截断。
- `info` 为空：视为有效的空配置并使用同一兜底。
- 单条语言为空或 `max_length <= 0`：忽略该条，其他有效条目继续使用。
- 分类 ID 不可用：不调用接口，记录告警并使用兜底。

## 测试策略

按测试驱动方式实现，并覆盖：

- product API 正确发送分类 ID 并解析多语言上限；
- 配置语言键规范化，非法条目被忽略；
- 产品名按英文、简体中文和繁体中文的动态上限截断；
- SKC 名按动态英文上限截断，AI 提示包含动态上限；
- 中文截断按 Unicode 字符数而非字节数；
- API 失败时英文使用 150，其他语言保持现有行为；
- 产品名与 SKC 名消费同一份任务级配置，不产生重复请求。

## 验收标准

- 分类 1772 返回示例配置时，英文、简体中文和繁体中文产品名/SKC 名分别不超过 150、100、105 个字符。
- 配置接口失败不导致任务失败，并有可诊断告警。
- 现有名称生成与发布测试全部通过。
