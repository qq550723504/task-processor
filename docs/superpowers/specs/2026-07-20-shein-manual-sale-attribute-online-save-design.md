# SHEIN 手工销售属性在线保存

## 目标

保存 ListingKit 的 SHEIN 手工销售属性时，始终以 SHEIN 当前类目模板和属性值接口为准；本地任务仅在在线校验成功后更新。页面刷新后必须显示已保存的自定义文本值。

## 保存流程

1. 前端提交主规格、次规格及每个 SKC/SKU 的选择或手工文本。
2. 服务端获取当前 SHEIN 类目属性模板。模板请求失败时，终止保存且不修改任务。
3. 对下拉选择的 `attribute_value_id`，确认该值仍在当前模板且属于所选属性。
4. 对手工文本，调用 SHEIN 校验；当 SHEIN 要求创建自定义值时，创建后使用返回的真实 `attribute_value_id` 与关联关系。
5. 只有所有规格都获得当前 SHEIN 确认的真实 ID，才应用修订并持久化任务；任一失败返回具体错误。
6. 前端重新加载时，若已保存值是模板外的自定义值，将其回填到文本输入框；模板内值继续回填下拉选择。

## 边界与错误处理

- 不复用本地 `SKCValueAssignments` / `SKUValueAssignments` 来绕过本次在线校验。
- 已选下拉值不再被视为天然有效，必须随本次保存请求对照最新模板。
- SHEIN 模板、校验或创建接口失败时，不追加成功修订记录，不更新资料包，并向用户呈现失败原因。
- 后端保存成功响应只代表已获取并写入 SHEIN 确认的值 ID，不代表已提交商品。

## 验收与测试

- 每次保存均调用模板读取接口。
- 下拉值在当前模板失效时保存失败且任务不变。
- 手工文本通过 SHEIN 校验/创建后写入返回的真实 ID 与关联关系。
- SHEIN 任一接口失败时不产生任务变更。
- 源值为 `1PCS`、已保存自定义值为 `11` 时，页面刷新后仍显示 `11`，再次保存提交 `11`。
