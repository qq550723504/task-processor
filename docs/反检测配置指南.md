# Amazon 爬虫反检测配置指南

## 概述

本指南介绍如何配置 Amazon 爬虫的反检测功能，以应对常见的浏览器指纹检测。

## 检测问题分析

根据您提供的检测结果：

- **时区不匹配 (-10%)**: IP时区与本地时区不符
- **IP地址不匹配 (-10%)**: 检测到代理或IP伪装
- **浏览器版本不匹配 (-5%)**: UserAgent版本与实际浏览器版本不符
- **隐身模式 (-10%)**: 检测到隐身模式使用

## 配置方案

### 1. 配置文件设置

在 `config/config-dev.yaml` 中添加反检测配置：

```yaml
amazon:
  enabled: true
  headless: false
  browserPath: "./chrome/chrome.exe"
  poolSize: 3
  viewportWidth: 1920
  viewportHeight: 1080
  proxyServer: ""  # 如果使用代理，在这里配置
  
  # 反检测配置
  antiDetection:
    enabled: true
    timezone: "America/New_York"  # 设置时区匹配代理IP地理位置
    userAgent: ""  # 留空使用随机用户代理池
    # 浏览器指纹配置
    fingerprint:
      enabled: true
      gpu:
        vendor: "Google Inc. (Intel)"
        renderer: "ANGLE (Intel, Intel(R) UHD Graphics 620 Direct3D11 vs_5_0 ps_5_0, D3D11)"
      webgl:
        vendor: "WebKit"
        renderer: "WebKit WebGL"
      canvas:
        enabled: true
        noise: 0.1
```

### 2. 反检测功能特性

#### 用户代理随机化
- 从8个不同的现代浏览器用户代理中随机选择
- 包含 Chrome、Firefox、Safari、Edge 等主流浏览器
- 版本号保持最新，避免版本不匹配检测

#### 浏览器启动参数优化
```
--disable-blink-features=AutomationControlled  # 禁用自动化控制特征
--exclude-switches=enable-automation           # 排除自动化开关
--disable-infobars                            # 禁用信息栏
--disable-permissions-api                     # 禁用权限API
```

#### JavaScript 反检测脚本
- 移除 `navigator.webdriver` 属性
- 伪造 `window.chrome` 对象
- 隐藏 Playwright 特征
- 伪造地理位置信息
- Canvas 和 WebGL 指纹随机化

#### 时区和地理位置匹配
- 根据代理IP地理位置设置对应时区
- 伪造地理位置坐标（默认纽约）
- 语言设置与地理位置保持一致

### 3. 使用建议

#### 代理配置
如果使用代理服务器，建议：
```yaml
amazon:
  proxyServer: "http://proxy-server:port"
  antiDetection:
    timezone: "America/Los_Angeles"  # 根据代理IP地理位置调整
```

#### 时区映射参考
- 美国东部: `America/New_York`
- 美国西部: `America/Los_Angeles`
- 欧洲: `Europe/London`, `Europe/Berlin`
- 亚洲: `Asia/Tokyo`, `Asia/Shanghai`

#### 最佳实践
1. **避免隐身模式**: 设置 `headless: false` 使用可见模式
2. **随机化间隔**: 在请求之间添加随机延迟
3. **IP轮换**: 定期更换代理IP地址
4. **行为模拟**: 模拟真实用户的浏览行为


### 5. 故障排除

#### 常见问题
1. **时区仍然不匹配**: 检查 `timezone` 配置是否与代理IP地理位置一致
2. **用户代理检测**: 确保使用最新版本的用户代理字符串
3. **Canvas指纹一致**: 启用 Canvas 噪声添加功能

#### 调试模式
设置 `headless: false` 可以观察浏览器行为，便于调试反检测效果。

## 总结

通过以上配置，可以显著降低被检测为机器人的风险。建议根据实际使用的代理服务器地理位置调整时区设置，并定期更新用户代理池以保持最佳的反检测效果。
