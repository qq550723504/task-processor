# TEMU平台属性处理架构重构计划

## 📋 重构概述

本次重构旨在解决当前属性处理系统中存在的架构问题，提升代码质量、性能和可维护性。

## 🎯 重构目标

### 主要目标
1. **解决架构债务**：拆分AIPropertyMapper的过重职责
2. **提升性能**：减少重复遍历，优化处理效率
3. **增强可维护性**：清晰的职责划分，统一的接口设计
4. **提高扩展性**：支持新属性类型和验证规则的快速扩展

### 预期收益
- 代码质量提升30%
- 处理性能提升20-30%
- 新功能开发效率提升40%
- Bug减少50%

## �️ 新架构设计

### 核心设计原则
1. **单一职责原则**：每个组件只负责一个明确的功能
2. **开闭原则**：对扩展开放，对修改关闭
3. **依赖倒置原则**：依赖抽象而不是具体实现
4. **管道模式**：通过管道串联各个处理阶段

### 新架构图
```
PropertyMappingOrchestrator (属性映射编排器)
    ↓
PropertyProcessingPipeline (属性处理管道)
    ├── PropertyMappingStage (映射阶段)
    │   ├── AIPropertyMapper (AI映射器)
    │   └── DefaultPropertyFiller (默认填充器)
    ├── PropertyValidationStage (验证阶段)
    │   └── ValidationRuleEngine (验证规则引擎)
    ├── PropertyFixingStage (修复阶段)
    │   └── PropertyFixerChain (属性修复链)
    └── PropertyFinalizationStage (最终化阶段)
        ├── PropertyDeduplicator (属性去重器)
        └── RequiredPropertyGuardian (必填属性守护者)
```

## 🔧 重构实施细节

### 1. PropertyProcessingPipeline设计

```go
// 属性处理管道接口
type PropertyProcessingPipeline interface {
    Process(ctx context.Context, templateProps []types.TemplateRespGoodsProperty, ext *models.ExtensionInfo) error
    AddStage(stage PropertyStage)
    GetStages() []PropertyStage
}

// 属性处理阶段接口
type PropertyStage interface {
    Process(ctx *PropertyContext) error
    GetName() string
    GetOrder() int
}

// 属性处理上下文
type PropertyContext struct {
    Context            context.Context
    TemplateProperties []types.TemplateRespGoodsProperty
    CurrentProperties  []models.PropertyItem
    Features          map[int]PropertyFeature
    Statistics        *ProcessingStats
    Logger            *logrus.Entry
    Cache             PropertyCache
}
```

### 2. 各阶段职责划分

#### PropertyMappingStage（映射阶段）
- **职责**：负责属性的初始映射
- **包含组件**：AIPropertyMapper, DefaultPropertyFiller
- **输入**：Amazon产品信息，TEMU模板属性
- **输出**：初始映射的属性列表

#### PropertyValidationStage（验证阶段）
- **职责**：验证属性的有效性和完整性
- **包含组件**：ValidationRuleEngine
- **输入**：初始映射的属性
- **输出**：验证结果和问题列表

#### PropertyFixingStage（修复阶段）
- **职责**：修复无效或不完整的属性
- **包含组件**：PropertyFixerChain
- **输入**：验证结果和问题列表
- **输出**：修复后的属性列表

#### PropertyFinalizationStage（最终化阶段）
- **职责**：最终的清理和保障工作
- **包含组件**：PropertyDeduplicator, RequiredPropertyGuardian
- **输入**：修复后的属性列表
- **输出**：最终的属性列表

### 3. 性能优化策略

#### 减少重复遍历
```go
// 一次性收集所有需要的信息
func (p *PropertyProcessingPipeline) buildContext(templateProps []types.TemplateRespGoodsProperty, ext *models.ExtensionInfo) *PropertyContext {
    ctx := &PropertyContext{
        TemplateProperties: templateProps,
        CurrentProperties:  ext.GoodsProperty.GoodsProperties,
        Features:          make(map[int]PropertyFeature),
        Statistics:        NewProcessingStats(),
    }

    // 一次性识别所有属性特征
    for _, templateProp := range templateProps {
        ctx.Features[templateProp.PID] = p.featureDetector.DetectFeatures(templateProp)
    }

    return ctx
}
```
