# 服务拆分部署指南

## 概述

项目已按功能拆分为独立的微服务，每个服务可以独立部署和扩展。

## 服务列表

### 1. TEMU 上架服务 (`temu-listing`)
- **功能**: 处理 TEMU 平台的商品上架任务
- **入口**: `cmd/temu-listing/main.go`
- **队列**: 监听 TEMU 相关的 RabbitMQ 队列
- **依赖**: RabbitMQ、管理系统 API、Amazon 爬虫（获取源商品信息）

### 2. SHEIN 上架服务 (`shein-listing`)
- **功能**: 处理 SHEIN 平台的商品上架任务
- **入口**: `cmd/shein-listing/main.go`
- **队列**: 监听 SHEIN 相关的 RabbitMQ 队列
- **依赖**: RabbitMQ、管理系统 API、Amazon 爬虫（获取源商品信息）

### 3. Amazon 爬虫服务 (`amazon-crawler`)
- **功能**: 从 Amazon 平台爬取商品信息
- **入口**: `cmd/amazon-crawler/main.go`
- **队列**: 监听爬虫任务队列，发布商品数据到结果队列
- **依赖**: RabbitMQ、Chrome/Chromium 浏览器

### 4. 1688 爬虫服务 (`1688-crawler`)
- **功能**: 从 1688 平台爬取商品信息
- **入口**: `cmd/1688-crawler/main.go`
- **队列**: 监听爬虫任务队列，发布商品数据到结果队列
- **依赖**: RabbitMQ、Chrome/Chromium 浏览器
- **状态**: 待实现

### 5. RabbitMQ Consumer (兼容模式)
- **功能**: 运行所有服务（上架 + 爬虫）
- **入口**: `cmd/rabbitmq-consumer/main.go`
- **用途**: 用于开发环境或单机部署场景

## 编译服务

## 部署方式

### 方式一：云电脑部署（当前架构）

适用于现有的 100 台云电脑环境。

#### 部署策略
1. **上架服务**: 部署在少数几台云电脑上（5-10台）
   - TEMU 上架服务: 3-5 台
   - SHEIN 上架服务: 3-5 台

2. **爬虫服务**: 部署在大部分云电脑上（90-95台）
   - Amazon 爬虫: 50-60 台
   - 1688 爬虫: 30-40 台

#### 启动命令
```bash
# TEMU 上架服务
./temu-listing --config=config/config-prod.yaml --log-level=info

# SHEIN 上架服务
./shein-listing --config=config/config-prod.yaml --log-level=info

# Amazon 爬虫服务
./amazon-crawler --config=config/config-prod.yaml --log-level=info

# 1688 爬虫服务
./1688-crawler --config=config/config-prod.yaml --log-level=info
```

#### 优势
- 资源利用率提升：爬虫服务可以充分利用云电脑的计算资源
- 灵活扩展：可以根据任务量动态调整各服务的实例数
- 故障隔离：单个服务故障不影响其他服务

### 方式二：Kubernetes 部署（推荐）

适用于有 K8s 集群的环境，详见 [K8s迁移实施方案.md](./K8s迁移实施方案.md)。

#### Deployment 配置示例

**TEMU 上架服务**:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: temu-listing
spec:
  replicas: 3
  selector:
    matchLabels:
      app: temu-listing
  template:
    metadata:
      labels:
        app: temu-listing
    spec:
      containers:
      - name: temu-listing
        image: your-registry/temu-listing:latest
        args:
          - "--config=/config/config-prod.yaml"
          - "--log-level=info"
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "2000m"
            memory: "2Gi"
```

**Amazon 爬虫服务**:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: amazon-crawler
spec:
  replicas: 10
  selector:
    matchLabels:
      app: amazon-crawler
  template:
    metadata:
      labels:
        app: amazon-crawler
    spec:
      containers:
      - name: amazon-crawler
        image: your-registry/amazon-crawler:latest
        args:
          - "--config=/config/config-prod.yaml"
          - "--log-level=info"
        resources:
          requests:
            cpu: "1000m"
            memory: "1Gi"
          limits:
            cpu: "4000m"
            memory: "4Gi"
```

#### HPA 自动扩缩容
```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: amazon-crawler-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: amazon-crawler
  minReplicas: 5
  maxReplicas: 50
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Pods
    pods:
      metric:
        name: rabbitmq_queue_messages
      target:
        type: AverageValue
        averageValue: "100"
```

## 配置文件

每个服务使用相同的配置文件格式，但只会使用相关的配置项。

### 配置示例 (`config/config-prod.yaml`)
```yaml
# RabbitMQ 配置（所有服务必需）
rabbitmq:
  enabled: true
  host: rabbitmq.example.com
  port: 5672
  username: ${RABBITMQ_USER}
  password: ${RABBITMQ_PASS}
  vhost: /

# 管理系统配置（上架服务必需）
management:
  base_url: https://api.example.com
  client_id: ${CLIENT_ID}
  client_secret: ${CLIENT_SECRET}
  tenant_id: ${TENANT_ID}

# 平台配置
platforms:
  temu:
    enabled: true
    # TEMU 相关配置...
  
  shein:
    enabled: true
    # SHEIN 相关配置...

# Amazon 配置（爬虫服务必需）
amazon:
  enabled: true
  # Amazon 爬虫配置...
```

## 日志管理

### 日志级别
- `debug`: 详细调试信息
- `info`: 一般信息（推荐生产环境）
- `warn`: 警告信息
- `error`: 错误信息

### 启动时指定日志级别
```bash
./temu-listing --log-level=debug
```

### 日志输出
- 标准输出（stdout）: 结构化 JSON 日志
- 文件输出: 可在配置文件中指定日志文件路径

## 故障排查

### 服务无法启动
1. 检查配置文件路径是否正确
2. 检查 RabbitMQ 连接配置
3. 查看日志输出的错误信息

### 服务运行但不处理任务
1. 检查 RabbitMQ 队列是否有消息
2. 检查服务是否正确订阅了队列
3. 查看服务日志，确认是否有错误

### 爬虫服务失败
1. 检查 Chrome/Chromium 是否正确安装
2. 检查浏览器驱动版本是否匹配
3. 查看爬虫日志，确认具体错误

## 性能优化建议

### 云电脑部署
1. **并发配置**: 根据云电脑配置（8核16G）调整并发数
   - 爬虫服务: 建议 4-6 个并发
   - 上架服务: 建议 2-4 个并发

2. **资源分配**: 
   - 爬虫服务占用资源较多，优先分配更多实例
   - 上架服务相对轻量，可以少量实例处理

3. **任务分配**: 
   - 使用 RabbitMQ 的公平分发（prefetch_count）
   - 避免单个实例过载

### K8s 部署
1. **HPA 配置**: 根据队列长度和 CPU 使用率自动扩缩容
2. **资源限制**: 设置合理的 requests 和 limits
3. **节点亲和性**: 爬虫服务可以调度到高配置节点

## 迁移路径

### 从单体到微服务
1. **阶段一**: 保持原有 `rabbitmq-consumer` 运行，同时部署新的微服务
2. **阶段二**: 逐步将流量切换到新服务
3. **阶段三**: 验证新服务稳定后，下线旧服务

### 从云电脑到 K8s
1. **阶段一**: 在 K8s 中部署少量实例，与云电脑并行运行
2. **阶段二**: 逐步增加 K8s 实例，减少云电脑实例
3. **阶段三**: 完全迁移到 K8s，释放云电脑资源
