## 🔧 系统架构

### 配置策略
1. **random**: 完全随机配置，每次生成不同的浏览器环境
2. **stable**: 基于实例ID的稳定配置，相同ID生成相同配置
3. **windows**: Windows专用配置，优化Windows环境
4. **preset**: 预设配置，使用预定义的高质量配置

### 预设配置
1. **windows_high_end**: Windows高端配置（RTX 4090, 16核）
2. **windows_mid_range**: Windows中端配置（GTX 1660 Ti, 8核）
3. **mac_high_end**: Mac高端配置（M2 Pro, 12核）

### 核心组件
- `ConfigManager`: 配置管理器，统一管理随机配置策略
- `RandomConfigGenerator`: 随机配置生成器，支持多平台多浏览器
- `BrowserPool`: 浏览器池，支持随机指纹和配置策略
- `InstanceManager`: 实例管理器，处理浏览器实例的创建和管理

## 🚀 使用示例

### 基本使用
```go
// 创建Amazon配置
amazonConfig := &config.AmazonConfig{
    Headless:       true,
    BrowserPath:    "chrome/chrome.exe", // fingerprint-chromium路径
    ViewportWidth:  1920,
    ViewportHeight: 1080,
}

// 创建浏览器池配置
poolConfig := &BrowserPoolConfig{
    Size:                5,
    UseRandomFingerprint: true,
    FingerprintStrategy:  "random", // 或 "stable", "windows", "preset"
    PresetName:          "windows_high_end",
}

// 创建和初始化浏览器池
pool := NewBrowserPool(amazonConfig, poolConfig)
err := pool.Initialize()
```

### 配置策略选择
```go
// 随机策略 - 每个实例都不同
configManager.GenerateBrowserConfig(cfg, "random", "", instanceID)

// 稳定策略 - 相同实例ID生成相同配置
configManager.GenerateBrowserConfig(cfg, "stable", "", instanceID)

// Windows专用策略 - 优化Windows环境
configManager.GenerateBrowserConfig(cfg, "windows", "", instanceID)

// 预设策略 - 使用高质量预设
configManager.GenerateBrowserConfig(cfg, "preset", "windows_high_end", instanceID)
```
