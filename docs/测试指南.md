# Go 测试指南

## 📋 概述

本文档提供项目中编写和运行测试的指南和最佳实践。

---

## 🎯 测试原则

### 1. 简洁优先
- 保持测试代码简单明了
- 避免过度使用 mock
- 专注于核心功能测试

### 2. 覆盖边界
- 测试正常情况
- 测试边界条件（0、负数、nil等）
- 测试错误情况

### 3. 快速反馈
- 测试应该快速运行
- 避免依赖外部服务
- 使用表驱动测试

---

## 📝 测试文件命名

### 规则
- 测试文件名：`xxx_test.go`
- 测试函数名：`TestXxx`
- 基准测试：`BenchmarkXxx`
- 示例函数：`ExampleXxx`

### 示例
```
cost_calculator.go       -> cost_calculator_test.go
url_helper.go           -> url_helper_test.go
manager_service.go      -> manager_service_test.go
```

## 🔧 测试类型

### 1. 单元测试

**目的：** 测试单个函数或方法

**示例：**
```go
func TestCalculateProductCost(t *testing.T) {
    calc := NewCostCalculator(nil, logger, false)
    
    result := calc.CalculateProductCost(100.0, 1, "P001", "SKU001")
    
    if result != 100.0 {
        t.Errorf("期望 100.0, 得到 %v", result)
    }
}
```

### 2. 表驱动测试

**目的：** 用多个测试用例测试同一个函数

**示例：**
```go
func TestCalculateProductCost(t *testing.T) {
    tests := []struct {
        name     string
        input    float64
        expected float64
    }{
        {"正常价格", 100.0, 100.0},
        {"零价格", 0, 0},
        {"负数价格", -10.0, 0},
    }
    
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result := calc.CalculateProductCost(tt.input, 1, "P001", "SKU001")
            if result != tt.expected {
                t.Errorf("期望 %v, 得到 %v", tt.expected, result)
            }
        })
    }
}
```

### 3. 基准测试

**目的：** 测试性能

**示例：**
```go
func BenchmarkCalculateProductCost(b *testing.B) {
    calc := NewCostCalculator(nil, logger, false)
    
    b.ResetTimer()
    for i := 0; i < b.N; i++ {
        calc.CalculateProductCost(100.0, 1, "P001", "SKU001")
    }
}
```

## 🚀 运行测试

### 运行所有测试
```bash
go test ./...
```

### 运行特定包的测试
```bash
go test ./internal/pkg/pricing/...
```

### 运行详细输出
```bash
go test ./... -v
```

### 运行特定测试
```bash
go test -run TestCalculateProductCost ./internal/pkg/pricing/
```

### 运行基准测试
```bash
go test -bench=. ./internal/pkg/pricing/
```


```bash
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out
```

## 💡 最佳实践

### 1. 使用子测试

**好：**
```go
func TestCalculate(t *testing.T) {
    t.Run("正常情况", func(t *testing.T) {
        // 测试代码
    })
    
    t.Run("边界情况", func(t *testing.T) {
        // 测试代码
    })
}
```

**不好：**
```go
func TestCalculateNormal(t *testing.T) { }
func TestCalculateBoundary(t *testing.T) { }
```

### 2. 测试边界条件

**必须测试：**
- nil 值
- 空字符串
- 零值
- 负数
- 最大值/最小值

**示例：**
```go
tests := []struct {
    name  string
    input float64
}{
    {"零值", 0},
    {"负数", -10.0},
    {"正常值", 100.0},
    {"小数", 99.99},
}
```

### 3. 使用 testify 断言（可选）

**安装：**
```bash
go get github.com/stretchr/testify
```

**使用：**
```go
import "github.com/stretchr/testify/assert"

func TestCalculate(t *testing.T) {
    result := Calculate(100.0)
    assert.Equal(t, 100.0, result)
    assert.NotNil(t, result)
}
```

### 4. 避免过度 Mock

**好：** 使用真实对象或简单的 nil
```go
calc := NewCostCalculator(nil, logger, false)
```

**不好：** 复杂的 mock 对象
```go
mockClient := &ComplexMockWithManyMethods{}
calc := NewCostCalculator(mockClient, logger, false)
```

### 5. 测试命名清晰

**好：**
```go
func TestCalculateProductCost_WithZeroPrice_ReturnsZero(t *testing.T)
```

**不好：**
```go
func TestCalc1(t *testing.T)
```

## 📊 性能测试指南

### 基准测试最佳实践

1. **使用 b.ResetTimer()**
```go
func BenchmarkXxx(b *testing.B) {
    // 准备工作
    setup()
    
    b.ResetTimer() // 重置计时器
    for i := 0; i < b.N; i++ {
        // 测试代码
    }
}
```

2. **避免编译器优化**
```go
var result float64

func BenchmarkXxx(b *testing.B) {
    var r float64
    for i := 0; i < b.N; i++ {
        r = Calculate(100.0)
    }
    result = r // 防止编译器优化掉计算
}
```

3. **测试不同输入大小**
```go
func BenchmarkCalculate(b *testing.B) {
    sizes := []int{10, 100, 1000, 10000}
    for _, size := range sizes {
        b.Run(fmt.Sprintf("size-%d", size), func(b *testing.B) {
            for i := 0; i < b.N; i++ {
                Calculate(size)
            }
        })
    }
}
```

### 性能指标解读

```
BenchmarkCalculateProductCost-12    28492869    42.38 ns/op    48 B/op    1 allocs/op
```

- `28492869`：每秒操作次数
- `42.38 ns/op`：每次操作耗时
- `48 B/op`：每次操作分配的内存
- `1 allocs/op`：每次操作的内存分配次数

**优秀指标：**
- 耗时：< 100 ns/op
- 内存：< 100 B/op
- 分配：< 2 allocs/op

## 🐛 常见问题

### 1. 测试崩溃

**问题：** `panic: runtime error: invalid memory address`

**原因：** nil 指针访问

**解决：**
```go
// 添加 nil 检查
if obj == nil {
    return defaultValue
}
```

### 2. 测试不稳定

**问题：** 测试有时通过，有时失败

**原因：** 
- 依赖外部状态
- 并发问题
- 时间依赖

**解决：**
- 使用固定的测试数据
- 避免并发测试
- Mock 时间函数

### 3. 测试太慢

**问题：** 测试运行时间过长

**原因：**
- 依赖外部服务
- 大量数据处理
- 重复初始化

**解决：**
- Mock 外部服务
- 减少测试数据量
- 使用 setup/teardown

## ✅ 测试检查清单

在提交代码前，确保：

- [ ] 所有测试通过 (`go test ./...`)
- [ ] 添加了新功能的测试
- [ ] 测试覆盖了边界条件
- [ ] 测试命名清晰
- [ ] 没有跳过的测试 (`t.Skip`)
- [ ] 基准测试性能合理
- [ ] 测试代码简洁易懂
- [ ] 没有硬编码的测试数据
