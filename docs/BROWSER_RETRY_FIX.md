# 浏览器超时重试机制优化

## 问题描述

之前当Amazon爬虫遇到浏览器超时（如 `playwright: timeout: Timeout 30000ms exceeded`）时：
1. 任务立即失败
2. 浏览器实例被**异步**重建
3. 任务需要重新入队等待下次执行

这导致：
- 任务失败率高
- 处理效率低
- 需要额外的重试机制

## 解决方案

### 修改内容

#### 1. `common/amazon/processor.go` - `processWithPool` 方法

**新增功能：**
- 在任务执行过程中检测到严重错误（超时、风控等）时
- **同步**销毁并重建浏览器实例
- 在**当前任务**中自动重试一次
- 只有重试失败后才让任务失败

**重试逻辑：**
```go
maxRetries := 1  // 最多重试1次（总共尝试2次）

for attempt := 0; attempt <= maxRetries; attempt++ {
    // 获取浏览器实例
    // 处理产品
    
    // 如果遇到严重错误（超时、风控等）
    if 严重错误 {
        // 同步重建浏览器实例
        newInstance := RecreateInstanceSync(oldInstance)
        
        // 如果还有重试机会，继续循环
        if attempt < maxRetries {
            continue  // 重试
        }
        
        // 否则返回错误
        return error
    }
}
```

#### 2. `common/amazon/browser_pool.go` - 新增 `RecreateInstanceSync` 方法

**功能：**
- 同步关闭旧的浏览器实例
- 立即创建新的浏览器实例
- 更新实例池中的引用
- 返回新实例供立即使用

**与原有 `recreateInstance` 的区别：**
- `RecreateInstanceSync`: **同步**重建，用于任务内重试
- `recreateInstance`: **异步**重建，用于后台健康检查

## 效果

### 修改前
```
[13:13:41] 提取器执行失败: timeout
[13:13:41] 检测到浏览器实例被风控
[13:13:41] 开始重新创建浏览器实例 (异步)
[13:13:41] 任务处理失败 ❌
[13:13:41] 任务重新入队，等待下次执行
```

### 修改后
```
[13:13:41] 提取器执行失败: timeout
[13:13:41] 检测到浏览器实例出现严重错误
[13:13:41] 开始同步重新创建浏览器实例
[13:13:43] 成功重建浏览器实例
[13:13:43] 开始第 1 次重试处理产品
[13:13:50] 成功处理产品 ✅
```

## 优势

1. **提高成功率**：单次超时不会导致任务失败
2. **减少延迟**：不需要等待任务重新入队
3. **资源利用**：立即重建实例，避免资源浪费
4. **用户体验**：减少任务失败通知

## 配置

当前配置为最多重试 **1次**（总共尝试2次）。如需调整，修改 `processWithPool` 中的：
```go
maxRetries := 1  // 可调整为 2, 3 等
```

## 产品不存在检测（新增）

### 问题
访问不存在的产品时，页面会超时，但这不是浏览器的问题，不应该触发浏览器重建。

### 解决方案

#### 1. 新增 `ProductNotFoundError` 错误类型
```go
type ProductNotFoundError struct {
    Message string
}
```

#### 2. 在 `processWithInstance` 中添加产品存在性检查
在提取产品信息之前，先检查：
- 是否显示"Page Not Found"
- 是否跳转到登录页面
- 是否显示验证码
- 是否存在产品标题元素

#### 3. 错误分类优化
- **产品不存在** → 返回 `ProductNotFoundError` → **不触发**浏览器重建
- **浏览器超时/风控** → 返回其他错误 → **触发**浏览器重建

### 效果

**产品不存在时：**
```
[13:42:17] 检测到产品不存在或页面异常
[13:42:17] 任务处理失败: 产品页面不存在 ❌
[13:42:17] 浏览器实例正常释放（不重建）
```

**浏览器超时时：**
```
[13:42:17] 检测到浏览器实例出现严重错误
[13:42:17] 同步重建浏览器实例
[13:42:19] 开始第 1 次重试
```

## 注意事项

1. 重建浏览器实例需要 2-5 秒时间
2. 如果重建失败，任务仍会失败
3. 只有严重错误（超时、风控等）才会触发重建重试
4. 产品不存在错误不会触发重建，直接返回错误
5. 普通错误（如元素未找到）不会触发重建，直接返回错误
