# 失败商品管理功能说明

## 功能概述

为了避免重复尝试上架已知失败的商品，系统新增了失败商品管理功能。当商品上架失败时，系统会自动记录失败的商品ID和相关信息到本地文件中，下次运行时会自动跳过这些商品。

## 功能特点

### 自动记录失败商品
- 当商品上架失败时，系统自动记录商品ID、SKU ID列表、失败原因、失败时间等信息
- 支持重试计数，记录每个商品的重试次数
- 按租户和店铺分别存储，避免数据混乱

### 智能跳过机制
- 下次运行时自动检查失败商品列表
- 跳过已知失败的商品，提高处理效率
- 在日志中明确标识跳过的原因

### 成功后自动清理
- 当之前失败的商品成功上架时，自动从失败列表中移除
- 确保失败列表的准确性

## 文件存储

失败商品记录存储在以下位置：
```
data/failed_goods/failed_goods_tenant_{租户ID}_store_{店铺ID}.json
```

例如：
- `data/failed_goods/failed_goods_tenant_1_store_508.json`

## 命令行选项

### 查看失败商品统计
```bash
./temu-bulk-relist -show-failed-stats
```
显示所有店铺的失败商品统计信息，包括：
- 失败商品总数
- 按失败原因分组统计
- 按重试次数分组统计

### 清理旧记录
```bash
./temu-bulk-relist -clean-failed-days=7
```
清理超过7天的失败记录，避免失败列表过大。

### 打印详细商品数据
```bash
./temu-bulk-relist -print-product-data=true -store=508
```
打印成功和失败商品的详细原始数据，用于分析哪些字段可能导致上架失败。

### 重置失败列表
```bash
./temu-bulk-relist -reset-failed-list
```
清空所有失败商品记录，重新开始记录。

## 商品数据分析功能

### 启用详细数据打印
使用 `-print-product-data=true` 选项可以打印每个成功和失败商品的详细原始数据，包括：

**基本信息:**
- 商品ID、商品名称、SKU ID
- 库存、普通库存、价格、货币

**分类信息:**
- 分类ID、分类列表、分类类型

**状态信息:**
- 惩罚标签、商品是否在售
- 状态4、子状态4、显示子状态4
- 需要整改、锁定状态

**其他属性:**
- 多站点商品、个性化状态、发货模式
- 库存显示标签、低流量标签、限制流量标签
- 易获利标签、是否为书籍、关闭类型列表

### 数据分析建议
通过对比成功和失败商品的数据，可以分析出可能导致上架失败的字段：

1. **惩罚标签 (punish_tags)**: 值大于0可能表示商品被惩罚
2. **需要整改 (need_rectification)**: true表示需要分类整改
3. **锁定状态 (allow_operate)**: false表示不允许上架操作
4. **库存相关**: 库存为0或普通库存不足
5. **状态字段**: status4_vo、sub_status4_vo等状态异常
6. **分类问题**: 某些分类可能被限制上架
7. **关闭类型**: closed_type_list不为空表示有关闭原因

## 记录格式

每条失败记录包含以下信息：

```json
{
  "goods_id": "603261834772989",
  "sku_ids": ["46749779528351"],
  "store_id": 508,
  "tenant_id": 1,
  "fail_reason": "请求体=&{GoodsID:603261834772989 SkuIDs:[46749779528351]}",
  "fail_time": "2026-01-11T17:00:17.9188331+08:00",
  "retry_count": 1,
  "last_retry": "0001-01-01T00:00:00Z"
}
```

字段说明：
- `goods_id`: 商品ID
- `sku_ids`: SKU ID列表
- `store_id`: 店铺ID
- `tenant_id`: 租户ID
- `fail_reason`: 失败原因
- `fail_time`: 首次失败时间
- `retry_count`: 重试次数
- `last_retry`: 最后重试时间

## 使用示例

### 基本使用
```bash
# 正常运行，系统会自动记录失败商品并跳过已知失败的商品
./temu-bulk-relist -store=508

# 查看失败商品统计
./temu-bulk-relist -show-failed-stats

# 清理30天前的失败记录
./temu-bulk-relist -store=508 -clean-failed-days=30
```

### 组合使用
```bash
# 处理所有店铺，同时清理旧记录
./temu-bulk-relist -all-stores -clean-failed-days=7 -verbose=true

# 分析商品数据，找出失败原因
./temu-bulk-relist -store=508 -print-product-data=true -verbose=true -log-file=analysis.log
```

## 注意事项

1. **文件权限**: 确保程序对 `data/failed_goods/` 目录有读写权限
2. **磁盘空间**: 长期运行可能积累大量失败记录，建议定期清理
3. **并发安全**: 失败商品管理器使用互斥锁保证并发安全
4. **异步保存**: 文件保存是异步进行的，不会阻塞主流程

## 故障排除

### 文件加载失败
如果看到 "解析失败商品记录失败" 的警告，可能是：
- JSON文件格式损坏
- 文件权限问题
- 磁盘空间不足

解决方法：
1. 检查文件内容是否为有效JSON
2. 删除损坏的文件，让系统重新创建
3. 使用 `-reset-failed-list` 重置失败列表

### 记录不生效
如果失败商品没有被正确跳过：
1. 检查租户ID和店铺ID是否正确
2. 确认文件路径是否正确
3. 查看日志中的错误信息
