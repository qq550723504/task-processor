## 🚀 使用示例

### 基本使用
```go
// 创建错误
err := errors.New(errors.ErrCodeValidation, "产品ID不能为空")

// 包装错误
err = errors.Wrap(err, errors.ErrCodeExternalAPI, "查询产品失败")

// 判断错误
if errors.IsRetryable(err) {
    // 重试
}
```

### 重试机制
```go
config := errors.DefaultRetryConfig()
err := errors.Retry(ctx, config, func() error {
    return callAPI()
})
```

### 错误处理器
```go
handler := errors.NewDefaultErrorHandler(logger)
err = handler.Handle(err)
if handler.ShouldRetry(err) {
    // 重试逻辑
}
```

### 安全执行
```go
// 安全执行函数
err := errors.SafeExecute(func() error {
    return riskyOperation()
}, logger)

// 安全启动goroutine
errors.SafeGo(func() {
    doBackgroundWork()
}, logger)
```

## 🎓 最佳实践

### ✅ 推荐做法
1. 使用具体的错误码
2. 包装底层错误，保留错误链
3. 使用错误处理器统一处理
4. 使用重试机制处理临时错误
5. 使用 `SafeGo` 启动 goroutine
6. 只在 main 函数中使用 `Fatal`

### ❌ 避免做法
1. 不要在运行时使用 panic
2. 不要丢失原始错误
3. 不要重复记录错误
4. 不要使用通用错误码
5. 不要在非main函数中使用 Fatal
