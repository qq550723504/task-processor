# TEMU平台属性处理架构分析

## 📋 概述

TEMU平台的属性处理系统是一个复杂的多层架构，负责处理从Amazon产品到TEMU产品的属性映射、验证、修复和填充。本文档详细分析了整个属性处理的调用链和各组件的职责。

## 🏗️ 整体架构图

```
Pipeline Builder
    ↓
BuildSpuHandler
    ↓
SpuBuilder.BuildExtensionInfo()
    ↓
AIPropertyMapper.BuildGoodsProperties()
    ├── AI映射成功 → PropertyValueFixer → PropertyValidator
    └── AI映射失败 → DefaultPropertyFiller → ValidationRuleEngine
                        ↓
                RequiredPropertyGuardian
                        ↓
                PropertyDeduplicator
```

### 4. AI属性映射器（核心组件）

```go
// internal/platforms/temu/handlers/ai_property_mapper_core.go
type AIPropertyMapper struct {
    logger       *logrus.Entry
    openaiClient *openaiClient.Client

    // 注入的专职处理器
    aiService         *AIService
    propertyValidator *PropertyValidator
    defaultFiller     *DefaultPropertyFiller  // 包含验证引擎

    // 新增的严格验证组件
    valueValidator     *PropertyValueValidator
    valueFixer         *PropertyValueFixer
    statsCollector     *PropertyValidationStatsCollector
    deduplicator       *PropertyDeduplicator
    selectionValidator *PropertySelectionValidator
    propertyGuardian   *RequiredPropertyGuardian
}

func (m *AIPropertyMapper) BuildGoodsProperties(temuCtx *temucontext.TemuTaskContext, ext *models.ExtensionInfo) error {
    // 获取模板信息
    templateInfo, exists := GetTemplateInfoFromContext(temuCtx)
    if !exists {
        return nil
    }

    // 准备AI映射数据
    mappingData := preparePropertyMappingData(temuCtx, templateInfo.GoodsProperties)

    // 调用AI进行属性映射
    aiCtx, cancel := context.WithTimeout(temuCtx.GetContext(), 60*time.Second)
    defer cancel()

    mappedProperties, err := m.aiService.CallAIForPropertyMapping(aiCtx, mappingData)
    
    if err != nil {
        // AI映射失败 → 使用默认填充器
        m.logger.WithError(err).Error("❌ AI属性映射失败，使用默认值填充所有必填属性")
        m.defaultFiller.FillRequiredPropertiesWithDefaults(templateInfo.GoodsProperties, ext)
        m.verifyRequiredProperties(templateInfo.GoodsProperties, ext)
        return nil
    }

    // AI映射成功 → 严格验证和修复流程
    m.enrichPropertiesWithTemplateInfo(mappedProperties, templateInfo.GoodsProperties)
    
    // 启动统计收集
    m.statsCollector.StartValidation(len(mappedProperties))

    // 使用严格验证和修复系统
    validatedProperties := m.valueFixer.FixAllInvalidProperties(mappedProperties, templateInfo.GoodsProperties)

    // 最终验证和应用
    finalProperties := m.performFinalValidation(validatedProperties, templateInfo.GoodsProperties)
    
    // 选择约束验证和去重
    allProperties := append(ext.GoodsProperty.GoodsProperties, finalProperties...)
    constraintValidatedProperties := m.selectionValidator.ValidateSelectionConstraints(allProperties, templateInfo.GoodsProperties)
    ext.GoodsProperty.GoodsProperties = m.deduplicator.DeduplicateByPidOnly(constraintValidatedProperties)

    // 最终保障：必填属性检查
    return m.propertyGuardian.GuardAllRequiredProperties(templateInfo.GoodsProperties, ext)
}
```

```go
type DefaultPropertyFiller struct {
    logger           *logrus.Entry
    deduplicator     *PropertyDeduplicator
    validationEngine *ValidationRuleEngine  // 新增：验证规则引擎
}

func (f *DefaultPropertyFiller) FillRequiredPropertiesWithDefaults(templateProps []types.TemplateRespGoodsProperty, ext *models.ExtensionInfo) {
    f.logger.Info("🔧 开始为必填属性填充默认值")

    // 第一步：填充缺失的必填属性
    filledCount := 0
    for _, templateProp := range templateProps {
        if templateProp.Required && !f.isPropertyAlreadyFilled(templateProp, ext) {
            f.FillSingleRequiredProperty(templateProp, ext)
            filledCount++
        }
    }

    // 第二步：使用验证引擎验证和修复所有属性（包括Material等特殊属性）
    if err := f.validationEngine.ValidateAndFixAll(templateProps, ext); err != nil {
        f.logger.Warnf("⚠️ 属性验证修复过程中出现错误: %v", err)
    }

    // 第三步：去重处理
    ext.GoodsProperty.GoodsProperties = f.deduplicator.DeduplicateByPidOnly(ext.GoodsProperty.GoodsProperties)

    f.logger.Infof("✅ 默认值填充和去重完成，共填充 %d 个必填属性", filledCount)
}
```

### 6. 验证规则引擎（新架构核心）

```go
// internal/platforms/temu/handlers/validation_rule_engine.go
type ValidationRuleEngine struct {
    logger          *logrus.Entry
    featureDetector *PropertyFeatureDetector
    rules           []ValidationRule
}

func NewValidationRuleEngine(logger *logrus.Entry) *ValidationRuleEngine {
    engine := &ValidationRuleEngine{
        logger:          logger,
        featureDetector: NewPropertyFeatureDetector(logger),
        rules:           make([]ValidationRule, 0),
    }
    
    // 注册内置规则
    engine.RegisterRule(NewPercentageSumRule(logger))
    // 未来可以注册更多规则...
    
    return engine
}

func (e *ValidationRuleEngine) ValidateAndFixAll(templateProps []types.TemplateRespGoodsProperty, ext *models.ExtensionInfo) error {
    e.logger.Info("🔍 开始验证和修复属性...")

    // 第一步：识别所有属性特征
    features := e.featureDetector.DetectAllFeatures(templateProps)

    // 第二步：按PID分组现有属性
    propertyGroups := e.groupPropertiesByPID(ext.GoodsProperty.GoodsProperties)

    // 第三步：遍历模板属性，应用匹配的验证规则
    fixedCount := 0
    for _, templateProp := range templateProps {
        feature, exists := features[templateProp.PID]
        if !exists {
            continue
        }

        props := propertyGroups[templateProp.PID]
        if len(props) == 0 {
            continue
        }

        // 应用所有匹配的验证规则
        for _, rule := range e.rules {
            if rule.Matches(feature) {
                result := rule.Validate(props, templateProp)
                if !result.IsValid && result.CanAutoFix {
                    if err := rule.Fix(props, templateProp); err == nil {
                        fixedCount++
                        e.logger.Infof("✅ 自动修复成功 [%s]: %s", rule.GetRuleName(), templateProp.Name)
                    }
                }
            }
        }
    }

    e.logger.Infof("✅ 属性验证完成，自动修复了 %d 个问题", fixedCount)
    return nil
}
```

```go
// internal/platforms/temu/handlers/property_feature_detector.go
func (d *PropertyFeatureDetector) DetectFeatures(templateProp types.TemplateRespGoodsProperty) PropertyFeature {
    feature := PropertyFeature{
        IsRequired: templateProp.Required,
        PID:        templateProp.PID,
        Name:       templateProp.Name,
    }

    // 识别数值输入类型（ControlType=16）
    if templateProp.ControlType == 16 {
        feature.HasNumericInput = true
        
        // 获取单位信息
        if len(templateProp.ValueUnit) > 0 {
            feature.ValueUnit = templateProp.ValueUnit[0]
        } else if len(templateProp.ValueUnitDTOList) > 0 {
            feature.ValueUnit = templateProp.ValueUnitDTOList[0].ValueUnit
        }

        // 识别百分比总和属性：ControlType=16 且 ValueUnit 包含 "%"
        if d.isPercentageUnit(feature.ValueUnit) {
            feature.IsPercentageSum = true
            feature.RequiredSum = 100
            d.logger.Infof("🔍 识别到百分比总和属性: %s (PID=%d)", templateProp.Name, templateProp.PID)
        }
    }

    return feature
}
```

### 8. 百分比总和验证规则（Material问题解决）

```go
// internal/platforms/temu/handlers/percentage_sum_rule.go
func (r *PercentageSumRule) Matches(feature PropertyFeature) bool {
    return feature.IsPercentageSum
}

func (r *PercentageSumRule) Validate(props []*models.PropertyItem, templateProp types.TemplateRespGoodsProperty) RuleValidationResult {
    currentSum := r.calculateSum(props)
    
    if currentSum != r.targetSum {
        return RuleValidationResult{
            IsValid:       false,
            ErrorMessage:  fmt.Sprintf("百分比总和为%d%%，期望%d%%", currentSum, r.targetSum),
            CurrentValue:  currentSum,
            ExpectedValue: r.targetSum,
            CanAutoFix:    true,
        }
    }
    
    return RuleValidationResult{IsValid: true}
}

func (r *PercentageSumRule) Fix(props []*models.PropertyItem, templateProp types.TemplateRespGoodsProperty) error {
    currentSum := r.calculateSum(props)
    r.logger.Infof("🔧 修复百分比总和: %d%% -> %d%%", currentSum, r.targetSum)

    if len(props) == 1 {
        // 单个属性：直接设为100%
        return r.fixSingleProperty(props[0])
    }

    // 多个属性：按比例调整
    return r.fixMultipleProperties(props, currentSum)
}
```

## 🔧 详细组件职责分析

### 🎯 核心处理组件

#### 1. AIPropertyMapper（属性映射总控制器）
- **职责**: 统筹整个属性映射流程，协调各个子组件
- **核心功能**: 
  - 调用AI进行智能属性映射
  - AI失败时调用默认填充器
  - 协调验证、修复、去重等各个环节
  - 提供完整的属性处理生命周期管理
- **关键方法**: `BuildGoodsProperties()`, `enrichPropertiesWithTemplateInfo()`

#### 2. DefaultPropertyFiller（默认属性填充器）
- **职责**: 为必填属性提供默认值，集成验证引擎
- **核心功能**:
  - 填充缺失的必填属性
  - 集成ValidationRuleEngine进行智能验证和修复
  - 属性去重处理
  - 支持多种属性类型的默认值生成
- **关键方法**: `FillRequiredPropertiesWithDefaults()`, `getDefaultValueForProperty()`

#### 3. ValidationRuleEngine（验证规则引擎）🆕
- **职责**: 通用验证规则引擎，支持可扩展的验证规则
- **核心功能**:
  - 基于属性特征识别需要验证的规则
  - 支持规则注册和动态扩展
  - 自动修复功能
  - 统计和监控验证过程
- **关键方法**: `ValidateAndFixAll()`, `RegisterRule()`

#### 4. PropertyFeatureDetector（属性特征识别器）🆕
- **职责**: 基于模板元数据识别属性特征，不依赖硬编码
- **核心功能**:
  - 识别百分比总和属性（ControlType=16 + ValueUnit="%"）
  - 识别数值输入类型属性
  - 识别必填属性
  - 支持扩展新的属性特征识别
- **关键方法**: `DetectFeatures()`, `DetectAllFeatures()`

### ✅ 验证组件群

#### 1. PropertyValidator（通用属性验证器）
- **职责**: 基础属性验证功能
- **功能**: 属性完整性验证、必填属性验证、基本约束检查

## 🎯 Material属性问题完整解决方案

### 问题根源分析
```json
// 原始问题：Material属性百分比不足100%
{
  "ref_pid": 2016,
  "pid": 2,
  "template_pid": 595935,
  "value": "Polyester",
  "vid": 46248,
  "value_unit": "%",
  "template_module_id": 48581,
  "number_input_value": "88"  // 只有88%，导致提交失败
}
```

### 解决方案实现路径
```
1. 属性特征识别阶段
PropertyFeatureDetector.DetectFeatures()
    ↓
检查 ControlType == 16 (数值输入选择类型)
    ↓
检查 ValueUnit 包含 "%" (百分比单位)
    ↓
标记为 IsPercentageSum = true, RequiredSum = 100

2. 验证规则匹配阶段
ValidationRuleEngine.ValidateAndFixAll()
    ↓
遍历所有注册的验证规则
    ↓
PercentageSumRule.Matches(feature) → 返回 true
    ↓
应用百分比总和验证规则

3. 验证阶段
PercentageSumRule.Validate()
    ↓
计算当前百分比总和: calculateSum() → 88%
    ↓
与目标值比较: 88% != 100%
    ↓
返回验证失败，CanAutoFix = true

4. 修复阶段
PercentageSumRule.Fix()
    ↓
检测到单个属性: len(props) == 1
    ↓
调用 fixSingleProperty()
    ↓
直接设置 number_input_value = "100"
    ↓
记录修复日志: "Polyester 88% -> 100%"
```

### 修复后的结果
```json
// 修复后：Material属性百分比为100%
{
  "ref_pid": 2016,
  "pid": 2,
  "template_pid": 595935,
  "value": "Polyester",
  "vid": 46248,
  "value_unit": "%",
  "template_module_id": 48581,
  "number_input_value": "100"  // 自动修复为100%
}
```

### 支持的修复场景
1. **单个材料**: 直接设为100%
2. **多个材料（有效值）**: 按比例调整，确保总和为100%
3. **多个材料（无效值）**: 平均分配，确保总和为100%
4. **混合场景**: 智能处理，保持合理的材料配比
